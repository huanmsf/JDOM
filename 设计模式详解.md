# Part 1: 设计模式概述

## 1.1 什么是设计模式

设计模式（Design Pattern）是软件开发人员在软件开发过程中面临的一般问题的解决方案。这些解决方案是众多软件开发人员经过相当长的一段时间的试验和错误总结出来的。

> **GoF（Gang of Four）**：1994年，Erich Gamma、Richard Helm、Ralph Johnson、John Vlissides 四人合著《设计模式：可复用面向对象软件的基础》，提出了23种经典设计模式。

### 为什么要学设计模式？

1. **提高代码复用性**：将重复出现的设计问题抽象为通用解决方案
2. **提高代码可读性**：使用设计模式后，代码结构更清晰，他人更易理解
3. **提高可维护性**：良好的设计使系统更易扩展和修改
4. **面试必考**：大厂面试高频考点，Spring/JDK源码中大量使用

## 1.3 面向对象七大设计原则

设计模式的基础是**面向对象七大设计原则**，掌握这些原则才能真正理解设计模式的精髓。

### 1.3.1 单一职责原则（SRP - Single Responsibility Principle）

> **定义**：一个类应该只有一个引起它变化的原因，即一个类只负责一项职责。

**反例（违反SRP）**：
```java
// 错误：一个类承担了多种职责
public class UserService {
    // 职责1：用户管理
    public void createUser(String name) { /*...*/ }
    public void deleteUser(Long id) { /*...*/ }
    
    // 职责2：发送邮件（不属于用户管理）
    public void sendEmail(String to, String content) { /*...*/ }
    
    // 职责3：数据库操作（不属于用户管理）
    public void saveToDatabase(Object obj) { /*...*/ }
}
```

**正例（遵循SRP）**：
```java
// 正确：每个类只负责一项职责
public class UserService {
    private EmailService emailService;
    private UserRepository userRepository;
    
    public void createUser(String name) {
        User user = new User(name);
        userRepository.save(user);
        emailService.sendWelcomeEmail(user.getEmail());
    }
}

public class EmailService {
    public void sendWelcomeEmail(String email) { /*...*/ }
}

public class UserRepository {
    public void save(User user) { /*...*/ }
}
```

---

### 1.3.2 开闭原则（OCP - Open/Closed Principle）

> **定义**：软件实体（类、模块、函数等）应该对**扩展开放**，对**修改关闭**。

**核心思想**：当需要添加新功能时，通过扩展现有代码来实现，而不是修改现有代码。

**反例（违反OCP）**：
```java
// 每次新增形状都要修改 AreaCalculator
public class AreaCalculator {
    public double calculateArea(Object shape) {
        if (shape instanceof Circle) {
            Circle c = (Circle) shape;
            return Math.PI * c.radius * c.radius;
        } else if (shape instanceof Rectangle) {
            Rectangle r = (Rectangle) shape;
            return r.width * r.height;
        }
        // 新增三角形 → 要修改这里！违反OCP
        throw new IllegalArgumentException("Unknown shape");
    }
}
```

**正例（遵循OCP）**：
```java
// 定义抽象，新增形状只需实现接口，无需修改 AreaCalculator
public interface Shape {
    double area();
}

public class Circle implements Shape {
    private double radius;
    public Circle(double radius) { this.radius = radius; }
    
    @Override
    public double area() { return Math.PI * radius * radius; }
}

public class Rectangle implements Shape {
    private double width, height;
    public Rectangle(double w, double h) { this.width = w; this.height = h; }
    
    @Override
    public double area() { return width * height; }
}

// 新增三角形：只需新建类，不修改任何已有代码
public class Triangle implements Shape {
    private double base, height;
    public Triangle(double b, double h) { this.base = b; this.height = h; }
    
    @Override
    public double area() { return 0.5 * base * height; }
}

public class AreaCalculator {
    public double calculateArea(Shape shape) {
        return shape.area(); // 对修改关闭，对扩展开放
    }
}
```

### 1.3.3 里氏替换原则（LSP - Liskov Substitution Principle）

> **定义**：子类对象应该能够替换父类对象，而不影响程序的正确性。

**经典反例 — 正方形继承矩形**：
```java
// 反例：正方形继承矩形违反LSP
public class Rectangle {
    protected int width, height;
    
    public void setWidth(int w) { this.width = w; }
    public void setHeight(int h) { this.height = h; }
    public int getArea() { return width * height; }
}

public class Square extends Rectangle {
    @Override
    public void setWidth(int w) {
        this.width = w;
        this.height = w; // 强制宽高相等
    }
    
    @Override
    public void setHeight(int h) {
        this.width = h;  // 强制宽高相等
        this.height = h;
    }
}

// 测试：子类无法替换父类
public void resize(Rectangle r) {
    r.setWidth(5);
    r.setHeight(10);
    // 对Rectangle期望面积=50，但Square得到100，LSP被违反！
    assert r.getArea() == 50; // Square会失败
}
```

**正例**：
```java
// 正确：提取公共抽象，不强行继承
public interface Shape {
    int getArea();
}

public class Rectangle implements Shape {
    private int width, height;
    // ... 独立实现
}

public class Square implements Shape {
    private int side;
    // ... 独立实现
}
```

### 1.3.4 依赖倒置原则（DIP - Dependency Inversion Principle）

> **定义**：高层模块不应该依赖低层模块，两者都应依赖其抽象；抽象不应依赖细节，细节应依赖抽象。

**反例（紧耦合）**：
```java
// 高层 OrderService 直接依赖低层 MySQLOrderRepository
public class OrderService {
    private MySQLOrderRepository repository = new MySQLOrderRepository(); // 硬依赖！
    
    public void createOrder(Order order) {
        repository.save(order); // 换成MongoDB？要改这里！
    }
}
```

**正例（依赖注入）**：
```java
// 定义抽象
public interface OrderRepository {
    void save(Order order);
    Order findById(Long id);
}

// 低层实现细节
public class MySQLOrderRepository implements OrderRepository {
    public void save(Order order) { /* MySQL实现 */ }
    public Order findById(Long id) { /* MySQL实现 */ return null; }
}

public class MongoOrderRepository implements OrderRepository {
    public void save(Order order) { /* MongoDB实现 */ }
    public Order findById(Long id) { /* MongoDB实现 */ return null; }
}

// 高层依赖抽象，通过构造器注入（Spring的核心思想）
public class OrderService {
    private final OrderRepository repository; // 依赖抽象接口
    
    public OrderService(OrderRepository repository) { // 依赖注入
        this.repository = repository;
    }
    
    public void createOrder(Order order) {
        repository.save(order); // 切换数据库只需换实现类，无需改此处
    }
}
```

> **Spring IoC容器**就是DIP原则的最佳实践：对象不创建依赖，而是由容器注入。

### 1.3.5 接口隔离原则（ISP - Interface Segregation Principle）

> **定义**：客户端不应该被迫依赖它不使用的接口方法。应将臃肿的接口拆分为多个专门的接口。

**反例（胖接口）**：
```java
// 胖接口：强迫实现所有方法
public interface Animal {
    void eat();
    void fly();  // 狗不会飞，却被迫实现
    void swim(); // 鸟不一定会游泳，却被迫实现
}

public class Dog implements Animal {
    public void eat() { System.out.println("Dog eating"); }
    public void fly() { throw new UnsupportedOperationException(); } // 被迫实现
    public void swim() { System.out.println("Dog swimming"); }
}
```

**正例（接口拆分）**：
```java
public interface Eatable { void eat(); }
public interface Flyable  { void fly(); }
public interface Swimmable{ void swim(); }

public class Dog implements Eatable, Swimmable {
    public void eat()  { System.out.println("Dog eating"); }
    public void swim() { System.out.println("Dog swimming"); }
    // 不实现Flyable，没有多余负担
}

public class Bird implements Eatable, Flyable {
    public void eat() { System.out.println("Bird eating"); }
    public void fly() { System.out.println("Bird flying"); }
}

public class Duck implements Eatable, Flyable, Swimmable {
    public void eat()  { System.out.println("Duck eating"); }
    public void fly()  { System.out.println("Duck flying"); }
    public void swim() { System.out.println("Duck swimming"); }
}
```

### 1.3.6 迪米特法则（LoD - Law of Demeter）

> **定义**：一个对象应该对其他对象有尽可能少的了解，也称"最少知道原则"。只与直接朋友通信。

**反例**：
```java
// 反例：OrderService 知道太多，深度调用链
public class OrderService {
    public double getOrderAmount(Order order) {
        // 违反LoD：通过多个中间对象获取数据
        return order.getCustomer().getWallet().getBalance();
    }
}
```

**正例**：
```java
// 正例：封装内部结构，提供直接接口
public class Order {
    private Customer customer;
    
    // Order封装对Customer的访问
    public double getCustomerBalance() {
        return customer.getBalance(); // 内部处理
    }
}

public class Customer {
    private Wallet wallet;
    
    public double getBalance() {
        return wallet.getBalance(); // 内部处理
    }
}

public class OrderService {
    public double getOrderAmount(Order order) {
        return order.getCustomerBalance(); // 只与直接朋友(Order)交互
    }
}
```

### 1.3.7 合成复用原则（CRP - Composite Reuse Principle）

> **定义**：优先使用对象组合/聚合，而不是继承来达到代码复用的目的。

**继承的缺点**：
- 破坏封装性（子类可访问父类实现细节）
- 继承是静态的，运行时无法改变
- "脆弱的基类"问题：修改父类可能破坏子类

**反例（滥用继承）**：
```java
// 反例：通过继承复用数据库操作
public class UserService extends BaseDBService {
    // 继承了所有DB操作，但可能只需要部分功能
}
```

**正例（组合复用）**：
```java
// 正例：通过组合复用
public class UserService {
    private final UserRepository repository;   // 组合：数据访问
    private final CacheService cacheService;   // 组合：缓存服务
    private final EmailService emailService;   // 组合：邮件服务
    
    public UserService(UserRepository repo, CacheService cache, EmailService email) {
        this.repository = repo;
        this.cacheService = cache;
        this.emailService = email;
    }
    
    // 灵活组合各种服务，运行时可替换实现
}
```

### 七大原则总结

| 原则 | 简记 | 核心思想 |
|------|------|---------|
| 单一职责原则(SRP) | 一个类，一个责任 | 高内聚 |
| 开闭原则(OCP) | 扩展开放，修改关闭 | 用抽象构建框架 |
| 里氏替换原则(LSP) | 子类可替换父类 | 继承要规范 |
| 依赖倒置原则(DIP) | 依赖抽象，不依赖具体 | IoC/DI基础 |
| 接口隔离原则(ISP) | 接口要小而专 | 低耦合 |
| 迪米特法则(LoD) | 最少知道 | 减少耦合 |
| 合成复用原则(CRP) | 优先组合，少用继承 | 灵活复用 |

# Part 2: 创建型模式（5种）

> 创建型模式关注**对象的创建机制**，将对象的创建与使用分离，使系统更灵活、独立于具体的对象创建方式。

---

## 2.1 单例模式 Singleton

### 定义与意图

> **确保一个类只有一个实例**，并提供一个全局访问点。

### UML类图

```
┌──────────────────────────────┐
│         Singleton            │
├──────────────────────────────┤
│ - instance: Singleton        │
├──────────────────────────────┤
│ - Singleton()                │
│ + getInstance(): Singleton   │
│ + doSomething(): void        │
└──────────────────────────────┘
```

### 应用场景

- 配置文件管理（全局唯一配置对象）
- 数据库连接池（避免重复创建连接）
- 日志记录器（全局唯一日志对象）
- Spring中 `@Bean` 默认 singleton 作用域

### 8种实现方式

#### 实现1：饿汉式（线程安全，推荐）

```java
/**
 * 饿汉式单例：类加载时就创建实例
 * 优点：线程安全（JVM类加载机制保证）、实现简单
 * 缺点：类加载时就初始化，不能延迟加载；若从未使用则浪费内存
 */
public class HungrySingleton {
    // 类加载时就创建实例（static final 保证线程安全）
    private static final HungrySingleton INSTANCE = new HungrySingleton();
    
    // 私有构造器，防止外部实例化
    private HungrySingleton() {}
    
    public static HungrySingleton getInstance() {
        return INSTANCE;
    }
    
    public void doSomething() {
        System.out.println("HungrySingleton working...");
    }
}
```

#### 实现2：懒汉式（线程不安全，不推荐）

```java
/**
 * 懒汉式单例：第一次使用时才创建
 * 优点：延迟加载
 * 缺点：线程不安全！多线程环境可能创建多个实例
 */
public class LazySingleton {
    private static LazySingleton instance;
    
    private LazySingleton() {}
    
    // 线程不安全！两个线程同时进入，可能各自创建一个实例
    public static LazySingleton getInstance() {
        if (instance == null) {          // 线程A、B同时到这里
            instance = new LazySingleton(); // 可能各自new一个
        }
        return instance;
    }
}
```

#### 实现3：同步方法懒汉式（线程安全但性能差）

```java
/**
 * 同步方法懒汉式
 * 优点：线程安全
 * 缺点：每次获取实例都要加锁，性能开销大（99.9%的调用是不需要加锁的）
 */
public class SyncMethodSingleton {
    private static SyncMethodSingleton instance;
    
    private SyncMethodSingleton() {}
    
    // synchronized 加到整个方法上，性能差
    public static synchronized SyncMethodSingleton getInstance() {
        if (instance == null) {
            instance = new SyncMethodSingleton();
        }
        return instance;
    }
}
```

#### 实现4：DCL双重检查锁（推荐，高性能线程安全）

```java
/**
 * DCL（Double-Checked Locking）双重检查锁
 * 优点：线程安全 + 延迟加载 + 高性能
 * 注意：volatile 关键字不可缺少！防止指令重排序
 */
public class DCLSingleton {
    // volatile 防止指令重排序：
    // new DCLSingleton() 分3步：1.分配内存 2.初始化对象 3.引用指向内存
    // 没有volatile，2和3可能重排序，导致另一线程拿到未初始化的对象
    private static volatile DCLSingleton instance;
    
    private DCLSingleton() {}
    
    public static DCLSingleton getInstance() {
        if (instance == null) {              // 第一次检查（无锁）：已初始化后不加锁
            synchronized (DCLSingleton.class) {
                if (instance == null) {      // 第二次检查（加锁）：防止重复创建
                    instance = new DCLSingleton();
                }
            }
        }
        return instance;
    }
}
```

**为什么需要volatile？**
```
// new DCLSingleton() 编译后的字节码：
1: memory = allocate();     // 分配内存空间
2: initInstance(memory);    // 初始化对象（可能被重排到3后面）
3: instance = memory;       // 引用指向内存

// 若指令重排为 1→3→2：
// 线程A执行到步骤3后被切出，instance != null 但对象未初始化
// 线程B第一次检查 instance != null，直接返回未初始化的对象！→ NPE
// volatile 禁止2和3的重排序，解决此问题
```

#### 实现5：静态内部类（推荐，最优雅）

```java
/**
 * 静态内部类单例（Initialization-on-demand holder idiom）
 * 优点：线程安全（JVM类加载保证）+ 延迟加载 + 无锁性能好
 * 原理：静态内部类只有在被引用时才加载，JVM保证类加载线程安全
 */
public class InnerClassSingleton {
    
    private InnerClassSingleton() {}
    
    // 静态内部类：只有调用 getInstance() 时才会加载此类
    private static class SingletonHolder {
        // JVM 类初始化时保证线程安全（Class Loading机制）
        private static final InnerClassSingleton INSTANCE = new InnerClassSingleton();
    }
    
    public static InnerClassSingleton getInstance() {
        return SingletonHolder.INSTANCE; // 触发 SingletonHolder 类加载
    }
    
    // 防止反序列化破坏单例
    protected Object readResolve() {
        return SingletonHolder.INSTANCE;
    }
}
```

#### 实现6：枚举单例（最安全，Effective Java推荐）

```java
/**
 * 枚举单例 - Joshua Bloch 在《Effective Java》中推荐
 * 优点：
 * 1. 天然线程安全（JVM枚举实现保证）
 * 2. 防止反序列化攻击（枚举反序列化不会创建新实例）
 * 3. 防止反射攻击（反射无法创建枚举实例）
 * 4. 代码最简洁
 */
public enum EnumSingleton {
    INSTANCE;
    
    // 单例的业务方法
    private int count = 0;
    
    public void doSomething() {
        count++;
        System.out.println("EnumSingleton called " + count + " times");
    }
    
    public static void main(String[] args) {
        EnumSingleton s1 = EnumSingleton.INSTANCE;
        EnumSingleton s2 = EnumSingleton.INSTANCE;
        System.out.println(s1 == s2); // true
    }
}
```

#### 实现7：容器式单例（Spring IoC风格）

```java
/**
 * 容器式单例：统一管理多种单例对象
 * 这是Spring IoC容器管理Bean的核心思想
 */
public class SingletonRegistry {
    private static final Map<String, Object> registry = new ConcurrentHashMap<>();
    
    private SingletonRegistry() {}
    
    @SuppressWarnings("unchecked")
    public static <T> T getInstance(String key, Class<T> clazz) {
        return (T) registry.computeIfAbsent(key, k -> {
            try {
                return clazz.getDeclaredConstructor().newInstance();
            } catch (Exception e) {
                throw new RuntimeException("创建实例失败: " + clazz.getName(), e);
            }
        });
    }
    
    public static void register(String key, Object instance) {
        registry.putIfAbsent(key, instance);
    }
    
    // 使用示例
    public static void main(String[] args) {
        UserService s1 = getInstance("userService", UserService.class);
        UserService s2 = getInstance("userService", UserService.class);
        System.out.println(s1 == s2); // true
    }
}
```

#### 实现8：ThreadLocal线程级单例

```java
/**
 * ThreadLocal单例：每个线程有自己的单例实例
 * 场景：数据库连接、用户会话信息等线程隔离的资源
 */
public class ThreadLocalSingleton {
    // 每个线程持有独立的实例
    private static final ThreadLocal<ThreadLocalSingleton> threadLocal =
        ThreadLocal.withInitial(ThreadLocalSingleton::new);
    
    private ThreadLocalSingleton() {}
    
    public static ThreadLocalSingleton getInstance() {
        return threadLocal.get();
    }
    
    // 使用完毕必须调用，防止内存泄漏！
    public static void remove() {
        threadLocal.remove();
    }
    
    public static void main(String[] args) throws InterruptedException {
        Thread t1 = new Thread(() -> {
            ThreadLocalSingleton s1 = getInstance();
            ThreadLocalSingleton s2 = getInstance();
            System.out.println("t1: s1==s2 ? " + (s1 == s2)); // true（同一线程）
        });
        
        Thread t2 = new Thread(() -> {
            ThreadLocalSingleton s = getInstance();
            System.out.println("t2 instance: " + s); // 与t1的实例不同
        });
        
        t1.start(); t2.start();
        t1.join(); t2.join();
    }
}
```

## 2.2 工厂方法模式 Factory Method

### 定义与意图

> **定义一个用于创建对象的接口，让子类决定实例化哪一个类**。工厂方法使一个类的实例化延迟到其子类。

### UML类图

```
┌─────────────────────┐         ┌─────────────────────┐
│   <<interface>>     │         │   <<interface>>      │
│      Creator        │         │      Product         │
├─────────────────────┤         ├─────────────────────┤
│ + createProduct()   │────────>│ + use(): void        │
│ + operation()       │         └─────────────────────┘
└─────────────────────┘                  △
          △                              │
          │                    ┌─────────────────────┐
   ┌──────┴──────┐             │   ConcreteProduct   │
   │             │             ├─────────────────────┤
ConcreteCreatorA ConcreteCreatorB │ + use(): void    │
   (创建ProductA) (创建ProductB) └─────────────────────┘
```

### 完整代码示例

```java
// 具体产品B

    @Override
    public void log(String message) {
        System.out.println("[CONSOLE] INFO: " + message);
    }
    
    @Override
    public void error(String message) {
        System.err.println("[CONSOLE] ERROR: " + message);
    }
}

// 具体产品C
public class DatabaseLogger implements Logger {
    private final String datasource;
    
    public DatabaseLogger(String datasource) {
        this.datasource = datasource;
    }
    
    @Override
    public void log(String message) {
        System.out.println("[DB:" + datasource + "] INFO: " + message);
    }
    
    @Override
    public void error(String message) {
        System.out.println("[DB:" + datasource + "] ERROR: " + message);

}

// 抽象工厂（Creator）
public abstract class LoggerFactory {
    // 工厂方法：子类实现，决定创建哪种Logger

    // 模板方法：使用工厂方法创建的产品

    }
}

// 具体工厂A
public class FileLoggerFactory extends LoggerFactory {
    private final String filename;
    
    public FileLoggerFactory(String filename) {
        this.filename = filename;
    }
    
    @Override
    public Logger createLogger() {
        return new FileLogger(filename);
    }
}

// 具体工厂B
public class ConsoleLoggerFactory extends LoggerFactory {
    @Override
    public Logger createLogger() {
        return new ConsoleLogger();
    }
}

// 简单工厂（非GoF，但常见）
public class SimpleLoggerFactory {
    public static Logger createLogger(String type, String... args) {
        switch (type.toLowerCase()) {
            case "file":
                return new FileLogger(args.length > 0 ? args[0] : "app.log");
            case "console":
                return new ConsoleLogger();
            case "database":
                return new DatabaseLogger(args.length > 0 ? args[0] : "default");
            default:
                throw new IllegalArgumentException("Unknown logger type: " + type);
        }
    }
}

// 客户端使用
public class Client {
    public static void main(String[] args) {
        // 工厂方法模式
        LoggerFactory factory = new FileLoggerFactory("app.log");
        Logger logger = factory.createLogger();
        logger.log("Application started");
        factory.logWithTimestamp("User logged in");
        
        // 简单工厂
        Logger consoleLogger = SimpleLoggerFactory.createLogger("console");
        consoleLogger.log("Simple factory example");
        
        // 切换实现只需换工厂，业务代码不变
        LoggerFactory consoleFactory = new ConsoleLoggerFactory();
        Logger logger2 = consoleFactory.createLogger();
        logger2.log("Switching to console logger");
    }
}
```

## 2.3 抽象工厂模式 Abstract Factory

### 定义与意图

> **提供一个创建一系列相关或相互依赖对象的接口**，而无需指定它们具体的类。

**工厂方法 vs 抽象工厂**：
- 工厂方法：创建**一种**产品（一个产品等级结构）
- 抽象工厂：创建**一族**产品（多个产品等级结构，产品族）

### UML类图

```
        AbstractFactory
       /               \
ConcreteFactory1   ConcreteFactory2
  createA() →A1     createA() →A2
  createB() →B1     createB() →B2

产品族：
  Factory1 → {A1, B1}  （同一风格/平台的一组产品）
  Factory2 → {A2, B2}
```

## 2.4 建造者模式 Builder

### 定义与意图

> **将一个复杂对象的构建与表示分离**，使同样的构建过程可以创建不同的表示。

**适用场景**：
- 对象有很多可选参数（避免"构造器爆炸"问题）
- 需要分步骤创建复杂对象
- 创建不可变对象

### UML类图

```
                    ┌─────────────────┐
                    │    Director     │
                    ├─────────────────┤
                    │ - builder       │
                    ├─────────────────┤
                    │ + construct()   │
                    └────────┬────────┘
                             │ uses
              ┌──────────────▼──────────────┐
              │    <<interface>> Builder     │
              ├─────────────────────────────┤
              │ + buildPartA()              │
              │ + buildPartB()              │
              │ + getResult()               │
              └──────────────┬──────────────┘
                             △
              ┌──────────────┴──────────────┐
              │      ConcreteBuilder         │
              ├─────────────────────────────┤
              │ - product: Product           │
              ├─────────────────────────────┤
              │ + buildPartA()              │
              │ + buildPartB()              │
              │ + getResult(): Product      │
              └─────────────────────────────┘
```

### 完整代码示例

```java
// 复杂产品：计算机

    private final String ram;
    private final String storage;
    
    // 可选参数
    private final String gpu;
    private final String cooler;
    private final String powerSupply;
    private final String case_;
    private final boolean wifi;
    private final boolean bluetooth;
    
    // 私有构造器，只能通过Builder创建
    private Computer(Builder builder) {
        this.cpu = builder.cpu;
        this.ram = builder.ram;
        this.storage = builder.storage;
        this.gpu = builder.gpu;
        this.cooler = builder.cooler;
        this.powerSupply = builder.powerSupply;
        this.case_ = builder.case_;
        this.wifi = builder.wifi;
        this.bluetooth = builder.bluetooth;
    }
    
    @Override
    public String toString() {
        return "Computer{" +
            "cpu='" + cpu + '\'' +
            ", ram='" + ram + '\'' +
            ", storage='" + storage + '\'' +
            (gpu != null ? ", gpu='" + gpu + '\'' : "") +
            (cooler != null ? ", cooler='" + cooler + '\'' : "") +
            (powerSupply != null ? ", psu='" + powerSupply + '\'' : "") +
            ", wifi=" + wifi +
            ", bluetooth=" + bluetooth +
            '}';
    }
    
    // 内部Builder类（链式调用）
    public static class Builder {
        // 必须参数
        private final String cpu;
        private final String ram;
        private final String storage;
        
        // 可选参数（设默认值）
        private String gpu = null;
        private String cooler = "原装散热";
        private String powerSupply = "400W";
        private String case_ = "标准机箱";
        private boolean wifi = false;
        private boolean bluetooth = false;
        
        // 必须参数通过构造器传入
        public Builder(String cpu, String ram, String storage) {
            if (cpu == null || ram == null || storage == null) {
                throw new IllegalArgumentException("CPU、内存、存储为必须参数");
            }
            this.cpu = cpu;
            this.ram = ram;
            this.storage = storage;
        }
        
        // 可选参数通过setter返回Builder（链式调用）
        public Builder gpu(String gpu) {
            this.gpu = gpu;
            return this;
        }
        
        public Builder cooler(String cooler) {
            this.cooler = cooler;
            return this;
        }
        
        public Builder powerSupply(String psu) {
            this.powerSupply = psu;
            return this;
        }
        
        public Builder case_(String case_) {
            this.case_ = case_;
            return this;
        }
        
        public Builder wifi(boolean wifi) {
            this.wifi = wifi;
            return this;
        }
        
        public Builder bluetooth(boolean bluetooth) {
            this.bluetooth = bluetooth;
            return this;
        }
        
        public Computer build() {
            // 可以在这里加业务校验
            return new Computer(this);
        }
    }
    
    public static void main(String[] args) {
        // 办公电脑（简单配置）

            .bluetooth(true)
            .build();
        
        System.out.println("办公电脑: " + officePC);

    }
```

## 2.5 原型模式 Prototype

### 定义与意图

> **通过复制现有实例来创建新对象**，而不是通过new关键字从头创建。

**适用场景**：
- 对象创建成本高（需要大量计算或IO）
- 需要创建大量相似对象
- 想保存对象某时刻的状态（配合备忘录模式）

### 浅拷贝 vs 深拷贝

```
浅拷贝（Shallow Copy）：
  Original: [name="A", scores=[1,2,3]]  ←共享同一个scores数组
  Clone:    [name="A", scores=[1,2,3]]  ←修改scores会互相影响！

深拷贝（Deep Copy）：
  Original: [name="A", scores=[1,2,3]]  ←各自独立的scores数组
  Clone:    [name="A", scores=[1,2,3]]  ←修改不会互相影响
```

### 完整代码示例

```java
import java.util.ArrayList;
import java.util.List;

// 原型接口
public interface Prototype<T> {
    T deepClone();
}

// 具体原型：文档模板
public class DocumentTemplate implements Prototype<DocumentTemplate>, Cloneable {
    private String title;
    private String author;
    private List<String> sections;          // 引用类型，需要深拷贝
    private List<String> attachments;       // 引用类型，需要深拷贝
    private Map<String, String> metadata;   // 引用类型，需要深拷贝
    
    public DocumentTemplate(String title, String author) {
        this.title = title;
        this.author = author;
        this.sections = new ArrayList<>();
        this.attachments = new ArrayList<>();
        this.metadata = new HashMap<>();
    }
    
    public DocumentTemplate addSection(String section) {
        this.sections.add(section);
        return this;
    }
    
    public DocumentTemplate addAttachment(String attachment) {
        this.attachments.add(attachment);
        return this;
    }
    
    public DocumentTemplate setMeta(String key, String value) {
        this.metadata.put(key, value);
        return this;
    }
    
    // 方法1：实现 Cloneable 的浅拷贝（不推荐）
    @Override
    public DocumentTemplate clone() {
        try {
            // Object.clone() 是浅拷贝！sections/attachments/metadata 共享引用
            return (DocumentTemplate) super.clone();
        } catch (CloneNotSupportedException e) {
            throw new AssertionError("Should not happen");
        }
    }
    
    // 方法2：手动实现深拷贝（推荐）
    @Override
    public DocumentTemplate deepClone() {
        DocumentTemplate clone = new DocumentTemplate(this.title, this.author);
        // 对每个引用类型手动深拷贝
        clone.sections = new ArrayList<>(this.sections);
        clone.attachments = new ArrayList<>(this.attachments);
        clone.metadata = new HashMap<>(this.metadata);
        return clone;
    }
    
    // 方法3：序列化实现深拷贝（通用但性能较差）
    @SuppressWarnings("unchecked")
    public DocumentTemplate serializableClone() {
        try {
            ByteArrayOutputStream bos = new ByteArrayOutputStream();
            ObjectOutputStream oos = new ObjectOutputStream(bos);
            oos.writeObject(this);
            
            ByteArrayInputStream bis = new ByteArrayInputStream(bos.toByteArray());
            ObjectInputStream ois = new ObjectInputStream(bis);
            return (DocumentTemplate) ois.readObject();
        } catch (Exception e) {
            throw new RuntimeException("序列化克隆失败", e);
        }
    }
    
    public void setTitle(String title) { this.title = title; }
    public String getTitle() { return title; }
    
    @Override
    public String toString() {
        return "DocumentTemplate{title='" + title + "', author='" + author +
            "', sections=" + sections + ", attachments=" + attachments + "}";
    }
    
    public static void main(String[] args) {
        // 创建标准合同模板（成本高，只需创建一次）
        DocumentTemplate template = new DocumentTemplate("标准合同", "法务部")
            .addSection("甲方信息")
            .addSection("乙方信息")
            .addSection("合同条款")
            .addSection("违约责任")
            .addSection("签字盖章")
            .addAttachment("公司营业执照")
            .setMeta("version", "v2.0")
            .setMeta("department", "法务部");
        
        System.out.println("原始模板: " + template);
        
        // 使用原型模式快速创建具体合同（比从头创建快得多）
        DocumentTemplate contract1 = template.deepClone();
        contract1.setTitle("采购合同-2024-001");
        contract1.addSection("采购清单");
        
        DocumentTemplate contract2 = template.deepClone();
        contract2.setTitle("服务协议-2024-002");
        contract2.addSection("服务范围");
        
        System.out.println("合同1: " + contract1);
        System.out.println("合同2: " + contract2);
        System.out.println("原始模板不变: " + template); // 验证模板未被修改
    }
}
```

## 3.1 适配器模式 Adapter

### 定义与意图

> **将一个类的接口转换成客户希望的另外一个接口**。适配器模式使得原本由于接口不兼容而不能一起工作的那些类可以一起工作。

### UML类图

```
客户端                   目标接口              适配器               被适配者
Client ─────────────> <<Target>>  <────── Adapter ─────────> Adaptee
                       + request()          + request()      + specificRequest()
                                            (调用adaptee.specificRequest())
```

## 3.2 装饰器模式 Decorator

### 定义与意图

> **动态地给一个对象添加一些额外的职责**，就增加功能来说，装饰器模式比生成子类更灵活。

**vs 继承**：继承是静态的（编译期确定），装饰器是动态的（运行时组合）。

### UML类图

```
<<Component>>
  + operation()
       △
       │
┌──────┴────────────────┐
│                       │
ConcreteComponent    Decorator
+ operation()        - component: Component
                     + Decorator(Component)
                     + operation() → component.operation()
                          △
                          │
               ┌──────────┴──────────┐
        ConcreteDecoratorA    ConcreteDecoratorB
        + operation()          + operation()
        (增加额外行为A)          (增加额外行为B)
```

## 3.3 代理模式 Proxy

### 定义与意图

> **为其他对象提供一种代理以控制对这个对象的访问**。

### 三种实现方式对比

```
┌──────────────┬────────────────────────────────────────────────────┐
│   实现方式    │  特点                                                │
├──────────────┼────────────────────────────────────────────────────┤
│ 静态代理      │ 编译期确定，需手写代理类，类多时维护困难              │
├──────────────┼────────────────────────────────────────────────────┤
│ JDK动态代理  │ 运行期生成，要求目标类实现接口，基于反射              │
├──────────────┼────────────────────────────────────────────────────┤
│ CGLIB代理    │ 运行期生成子类字节码，无需接口，基于继承              │
└──────────────┴────────────────────────────────────────────────────┘
```

### 实现2：JDK动态代理

```java
import java.lang.reflect.InvocationHandler;
import java.lang.reflect.Method;
import java.lang.reflect.Proxy;

// 通用的JDK动态代理处理器
public class LoggingInvocationHandler implements InvocationHandler {
    private final Object target; // 被代理的真实对象
    
    public LoggingInvocationHandler(Object target) {
        this.target = target;
    }
    
    @Override
    public Object invoke(Object proxy, Method method, Object[] args) throws Throwable {
        long start = System.currentTimeMillis();
        System.out.printf("[JDK代理] 调用 %s.%s, 参数: %s%n",
            target.getClass().getSimpleName(), method.getName(), java.util.Arrays.toString(args));
        
        try {
            Object result = method.invoke(target, args); // 反射调用真实方法
            long elapsed = System.currentTimeMillis() - start;
            System.out.printf("[JDK代理] %s 完成, 耗时: %dms, 返回: %s%n",
                method.getName(), elapsed, result);
            return result;
        } catch (Exception e) {
            System.out.printf("[JDK代理] %s 异常: %s%n", method.getName(), e.getCause().getMessage());
            throw e.getCause();
        }
    }
    
    // 工厂方法：创建代理实例
    @SuppressWarnings("unchecked")
    public static <T> T createProxy(T target, Class<T> interfaceClass) {
        return (T) Proxy.newProxyInstance(
            target.getClass().getClassLoader(),      // ClassLoader
            new Class[]{interfaceClass},              // 目标接口
            new LoggingInvocationHandler(target)      // 处理器
        );
    }
}

// 使用
public class JDKProxyDemo {
    public static void main(String[] args) {
        UserService realService = new UserServiceImpl();
        
        // 创建代理（运行时动态生成代理类）
        UserService proxy = LoggingInvocationHandler.createProxy(realService, UserService.class);
        
        // 调用代理
        proxy.findById(1L);
        proxy.createUser(new User(null, "张三"));
        
        System.out.println("代理类名: " + proxy.getClass().getName());
        // 输出类似: com.sun.proxy.$Proxy0
    }
}
```

```java
    // final方法不能被CGLIB代理！
    public final void systemMethod() {
        System.out.println("final方法，CGLIB无法拦截");
    }
}

// CGLIB方法拦截器
public class CglibLoggingInterceptor implements MethodInterceptor {
    @Override
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy) throws Throwable {
        System.out.printf("[CGLIB代理] 调用 %s, 参数: %s%n",
            method.getName(), java.util.Arrays.toString(args));
        
        long start = System.currentTimeMillis();
        // proxy.invokeSuper 调用父类（真实类）的方法，避免递归
        Object result = proxy.invokeSuper(obj, args);
        System.out.printf("[CGLIB代理] %s 耗时: %dms%n",
            method.getName(), System.currentTimeMillis() - start);
        return result;
    }
}

// CGLIB代理工厂
public class CglibProxyFactory {
    @SuppressWarnings("unchecked")
    public static <T> T createProxy(Class<T> targetClass) {
        Enhancer enhancer = new Enhancer();
        enhancer.setSuperclass(targetClass);               // 设置父类（目标类）
        enhancer.setCallback(new CglibLoggingInterceptor()); // 设置拦截器
        return (T) enhancer.create();                       // 创建代理（子类）实例
    }
}

// 使用
public class CglibProxyDemo {
    public static void main(String[] args) {
        UserManager proxy = CglibProxyFactory.createProxy(UserManager.class);
        proxy.findById(1L);     // 被拦截
        proxy.createUser(new User(null, "李四")); // 被拦截
        proxy.systemMethod();   // final方法，不被拦截
        
        System.out.println("代理类名: " + proxy.getClass().getName());
        // 输出类似: UserManager$$EnhancerByCGLIB$$xxxxx
    }
}
```

## 3.4 外观模式 Facade

### 定义与意图

## 3.5 桥接模式 Bridge

### 定义与意图

> **将抽象部分与它的实现部分分离**，使它们都可以独立地变化。

**核心问题**：当一个类有多个维度的变化时，避免类爆炸。

**类爆炸例子**：
```
形状×颜色（2维）：红圆、蓝圆、红方、蓝方 = 2×2=4类
形状×颜色×大小（3维）：= 2×2×2=8类  → 类数量指数级增长！
```

### 完整代码示例

```java
// 实现维度接口：颜色
public interface Color {
    String fill();
}

public class Red implements Color {
    @Override
    public String fill() { return "红色"; }
}

public class Blue implements Color {
    @Override
    public String fill() { return "蓝色"; }
}

public class Green implements Color {
    @Override
    public String fill() { return "绿色"; }
}

// 抽象维度：形状（持有实现维度的引用 = "桥"）
public abstract class Shape {
    protected Color color; // 桥接：持有实现维度的引用
    
    public Shape(Color color) {
        this.color = color;
    }
    
    public abstract void draw();
    
    // 运行时可以动态改变颜色！
    public void setColor(Color color) {
        this.color = color;
    }
}

public class Circle extends Shape {
    private double radius;
    
    public Circle(double radius, Color color) {
        super(color);
        this.radius = radius;
    }
    
    @Override
    public void draw() {
        System.out.printf("绘制%s圆形 (半径=%.1f)%n", color.fill(), radius);
    }
}

public class Rectangle extends Shape {
    private double width, height;
    
    public Rectangle(double width, double height, Color color) {
        super(color);
        this.width = width;
        this.height = height;
    }
    
    @Override
    public void draw() {
        System.out.printf("绘制%s矩形 (%.1f × %.1f)%n", color.fill(), width, height);
    }
}

public class Triangle extends Shape {
    private double base, height;
    
    public Triangle(double base, double height, Color color) {
        super(color);
        this.base = base;
        this.height = height;
    }
    
    @Override
    public void draw() {
        System.out.printf("绘制%s三角形 (底=%.1f, 高=%.1f)%n", color.fill(), base, height);
    }
}

// 客户端
public class BridgeDemo {
    public static void main(String[] args) {
        // 3种形状 × 3种颜色 = 9种组合，只用了 3+3=6 个类！
        Shape redCircle = new Circle(5.0, new Red());
        Shape blueRect = new Rectangle(4.0, 3.0, new Blue());
        Shape greenTri = new Triangle(3.0, 4.0, new Green());
        
        redCircle.draw();   // 绘制红色圆形 (半径=5.0)
        blueRect.draw();    // 绘制蓝色矩形 (4.0 × 3.0)
        greenTri.draw();    // 绘制绿色三角形 (底=3.0, 高=4.0)
        
        // 运行时动态切换颜色
        redCircle.setColor(new Blue());
        redCircle.draw();   // 绘制蓝色圆形 (半径=5.0)
    }
}
```

## 3.6 组合模式 Composite

### 定义与意图

> **将对象组合成树形结构以表示"部分-整体"的层次结构**。组合模式使得用户对单个对象和组合对象的使用具有一致性。

**典型场景**：文件系统、XML/HTML文档树、组织架构树、菜单树。

### 完整代码示例（文件系统）

```java
// 抽象组件
public abstract class FileSystemComponent {
    protected String name;
    
    public FileSystemComponent(String name) {
        this.name = name;
    }
    
    public String getName() { return name; }
    
    // 共同操作
    public abstract long getSize();
    public abstract void print(String indent);
    
    // 容器操作（叶子节点默认抛异常）
    public void add(FileSystemComponent component) {
        throw new UnsupportedOperationException(name + " 不支持add操作");
    }
    public void remove(FileSystemComponent component) {
        throw new UnsupportedOperationException(name + " 不支持remove操作");
    }
    public List<FileSystemComponent> getChildren() {
        throw new UnsupportedOperationException(name + " 没有子节点");
    }
}

// 叶子节点：文件
public class File extends FileSystemComponent {
    private long size;
    private String type;
    
    public File(String name, long size, String type) {
        super(name);
        this.size = size;
        this.type = type;
    }
    
    @Override
    public long getSize() { return size; }
    
    @Override
    public void print(String indent) {
        System.out.printf("%s📄 %s (%s, %d bytes)%n", indent, name, type, size);
    }
}

// 组合节点：文件夹
public class Directory extends FileSystemComponent {
    private List<FileSystemComponent> children = new ArrayList<>();
    
    public Directory(String name) {
        super(name);
    }
    
    @Override
    public void add(FileSystemComponent component) {
        children.add(component);
    }
    
    @Override
    public void remove(FileSystemComponent component) {
        children.remove(component);
    }
    
    @Override
    public List<FileSystemComponent> getChildren() {
        return Collections.unmodifiableList(children);
    }
    
    @Override
    public long getSize() {
        // 递归计算：总大小 = 所有子组件大小之和
        return children.stream().mapToLong(FileSystemComponent::getSize).sum();
    }
    
    @Override
    public void print(String indent) {
        System.out.printf("%s📁 %s/ (总计: %d bytes)%n", indent, name, getSize());
        for (FileSystemComponent child : children) {
            child.print(indent + "  "); // 递归打印，增加缩进
        }
    }
}

// 客户端（对文件和文件夹统一处理）
public class FileSystemDemo {
    public static void main(String[] args) {
        // 构建目录树
        Directory root = new Directory("project");
        
        Directory src = new Directory("src");
        Directory main = new Directory("main");
        Directory java = new Directory("java");
        java.add(new File("Main.java", 2048, "Java源码"));
        java.add(new File("App.java", 1536, "Java源码"));
        Directory resources = new Directory("resources");
        resources.add(new File("application.yml", 512, "配置文件"));
        main.add(java);
        main.add(resources);
        src.add(main);
        
        Directory test = new Directory("test");
        Directory testJava = new Directory("java");
        testJava.add(new File("MainTest.java", 1024, "测试文件"));
        test.add(testJava);
        src.add(test);
        
        root.add(src);
        root.add(new File("pom.xml", 3072, "Maven配置"));
        root.add(new File("README.md", 4096, "文档"));
        
        // 统一处理：打印树结构
        root.print("");
        
        System.out.println("\n项目总大小: " + root.getSize() + " bytes");
        
        // 对单个文件和文件夹使用相同的接口
        FileSystemComponent component = root; // 可以是File也可以是Directory
        System.out.println("根节点大小: " + component.getSize());
    }
}
```

## 3.7 享元模式 Flyweight

### 定义与意图

> **运用共享技术有效地支持大量细粒度对象**。通过共享已有对象来减少内存占用，提高性能。

**关键概念**：
- **内部状态（Intrinsic State）**：对象可共享的部分，不随环境变化
- **外部状态（Extrinsic State）**：对象随环境变化的部分，由客户端传入

### UML类图

```
Client ──> FlyweightFactory ──> Flyweight (共享)
                │                    △
                └──map缓存──>  ConcreteFlyweight
                                (内部状态)
                              + operation(extrinsicState)
                                        ↑
                               外部状态由客户端传入
```

### 完整代码示例（围棋棋子）

```java
// 享元对象：棋子（颜色是内部状态，可共享）
public class ChessPiece {
    // 内部状态：不变，可以共享
    private final String color; // "黑" 或 "白"
    private final String shape; // 棋子形状
    
    public ChessPiece(String color, String shape) {
        this.color = color;
        this.shape = shape;
    }
    
    // 外部状态（位置）由方法参数传入，不保存在对象中
    public void display(int x, int y) {
        System.out.printf("棋子[%s/%s] 落在位置(%d, %d)%n", color, shape, x, y);
    }
    
    public String getColor() { return color; }
}

// 享元工厂：管理享元对象池
public class ChessPieceFactory {
    private static final Map<String, ChessPiece> pool = new HashMap<>();
    
    static {
        // 预创建享元对象（只有黑白两种，整局游戏共享）
        pool.put("黑", new ChessPiece("黑", "●"));
        pool.put("白", new ChessPiece("白", "○"));
    }
    
    // 获取享元对象（从池中取，不重复创建）
    public static ChessPiece getChessPiece(String color) {
        ChessPiece piece = pool.get(color);
        if (piece == null) {
            throw new IllegalArgumentException("不支持的棋子颜色: " + color);
        }
        return piece;
    }
    
    public static int getPoolSize() { return pool.size(); }
}

// 棋局（管理棋子位置，外部状态）
public class ChessBoard {
    // 记录每颗棋子的位置（外部状态）
    private final List<int[]> blackPositions = new ArrayList<>();
    private final List<int[]> whitePositions = new ArrayList<>();
    
    public void placeBlack(int x, int y) {
        blackPositions.add(new int[]{x, y});
        // 获取共享的黑棋对象（不new新对象！）
        ChessPiece black = ChessPieceFactory.getChessPiece("黑");
        black.display(x, y);
    }
    
    public void placeWhite(int x, int y) {
        whitePositions.add(new int[]{x, y});
        ChessPiece white = ChessPieceFactory.getChessPiece("白");
        white.display(x, y);
    }
    
    public void printStats() {
        System.out.println("=== 棋局统计 ===");
        System.out.println("黑棋落子数: " + blackPositions.size());
        System.out.println("白棋落子数: " + whitePositions.size());
        System.out.println("棋子对象池大小: " + ChessPieceFactory.getPoolSize() 
            + " (无论落多少子，只有2个棋子对象！)");
    }
}

// 演示：内存节省效果
public class FlyweightDemo {
    public static void main(String[] args) {
        ChessBoard board = new ChessBoard();
        
        // 模拟100步棋
        Random rand = new Random(42);
        for (int i = 0; i < 50; i++) {
            board.placeBlack(rand.nextInt(19), rand.nextInt(19));
            board.placeWhite(rand.nextInt(19), rand.nextInt(19));
        }
        
        board.printStats();
        
        // 验证共享：两个黑棋引用是同一个对象
        ChessPiece b1 = ChessPieceFactory.getChessPiece("黑");
        ChessPiece b2 = ChessPieceFactory.getChessPiece("黑");
        System.out.println("b1 == b2 ? " + (b1 == b2)); // true！
    }
}
```

## 4.1 策略模式 Strategy

### 定义与意图

> **定义一系列的算法，把它们一个个封装起来，并且使它们可相互替换**。策略模式使算法可独立于使用它的客户端而变化。

**核心价值**：消除大量 if-else / switch-case，解耦算法与使用者。

### 完整代码示例（电商促销策略 + 消除if-else）

```java
// 策略接口
public interface DiscountStrategy {
    /**
     * 计算折后价格
     * @param originalPrice 原价
     * @param quantity 购买数量
     * @return 折后价格
     */
    BigDecimal calculate(BigDecimal originalPrice, int quantity);
    
    String getDescription();
}

// 策略A：无折扣
public class NoDiscountStrategy implements DiscountStrategy {
    @Override
    public BigDecimal calculate(BigDecimal originalPrice, int quantity) {
        return originalPrice.multiply(BigDecimal.valueOf(quantity));
    }
    
    @Override
    public String getDescription() { return "无折扣"; }
}

// 策略B：九折优惠
public class NinetyPercentStrategy implements DiscountStrategy {
    @Override
    public BigDecimal calculate(BigDecimal originalPrice, int quantity) {
        return originalPrice.multiply(BigDecimal.valueOf(quantity))
            .multiply(new BigDecimal("0.9"))
            .setScale(2, RoundingMode.HALF_UP);
    }
    
    @Override
    public String getDescription() { return "九折优惠"; }
}

// 策略C：满减（满100减20）
public class FullReductionStrategy implements DiscountStrategy {
    private final BigDecimal threshold;  // 满多少
    private final BigDecimal reduction;  // 减多少
    
    public FullReductionStrategy(BigDecimal threshold, BigDecimal reduction) {
        this.threshold = threshold;
        this.reduction = reduction;
    }
    
    @Override
    public BigDecimal calculate(BigDecimal originalPrice, int quantity) {
        BigDecimal total = originalPrice.multiply(BigDecimal.valueOf(quantity));
        // 计算可以减多少次
        long times = total.divideToIntegralValue(threshold).longValue();
        BigDecimal discount = reduction.multiply(BigDecimal.valueOf(times));
        return total.subtract(discount).max(BigDecimal.ZERO);
    }
    
    @Override
    public String getDescription() {
        return "满" + threshold + "减" + reduction;
    }
}

// 策略D：买N送1
public class BuyNGetOneFreeStrategy implements DiscountStrategy {
    private final int buyCount; // 买几件
    
    public BuyNGetOneFreeStrategy(int buyCount) {
        this.buyCount = buyCount;
    }
    
    @Override
    public BigDecimal calculate(BigDecimal originalPrice, int quantity) {
        // 买buyCount件送1件，实际付款数量
        int payCount = quantity - quantity / (buyCount + 1);
        return originalPrice.multiply(BigDecimal.valueOf(payCount));
    }
    
    @Override
    public String getDescription() { return "买" + buyCount + "送1"; }
}

// 策略E：VIP折扣（根据VIP等级动态计算）
public class VIPDiscountStrategy implements DiscountStrategy {
    private final int vipLevel; // VIP等级 1-5
    
    public VIPDiscountStrategy(int vipLevel) {
        this.vipLevel = Math.max(1, Math.min(5, vipLevel));
    }
    
    @Override
    public BigDecimal calculate(BigDecimal originalPrice, int quantity) {
        // VIP1: 95折, VIP2: 9折, VIP3: 85折, VIP4: 8折, VIP5: 75折
        double discount = 1.0 - (vipLevel * 0.05);
        return originalPrice.multiply(BigDecimal.valueOf(quantity))
            .multiply(BigDecimal.valueOf(discount))
            .setScale(2, RoundingMode.HALF_UP);
    }
    
    @Override
    public String getDescription() { return "VIP" + vipLevel + "折扣"; }
}

// ===== Context（策略使用者）=====
public class ShoppingCart {
    private DiscountStrategy strategy;
    private List<CartItem> items = new ArrayList<>();
    
    public ShoppingCart(DiscountStrategy strategy) {
        this.strategy = strategy;
    }
    
    // 运行时动态切换策略
    public void setStrategy(DiscountStrategy strategy) {
        this.strategy = strategy;
    }
    
    public void addItem(String name, BigDecimal price, int qty) {
        items.add(new CartItem(name, price, qty));
    }
    
    public void checkout() {
        System.out.println("=== 购物车结算 ===");
        System.out.println("促销策略: " + strategy.getDescription());
        
        BigDecimal total = BigDecimal.ZERO;
        for (CartItem item : items) {
            BigDecimal itemTotal = strategy.calculate(item.getPrice(), item.getQuantity());
            System.out.printf("  %s × %d = ¥%.2f (原价¥%.2f)%n",
                item.getName(), item.getQuantity(),
                itemTotal, item.getPrice().multiply(BigDecimal.valueOf(item.getQuantity())));
            total = total.add(itemTotal);
        }
        System.out.printf("应付总计: ¥%.2f%n", total);
    }
}

// ===== 策略工厂：彻底消除if-else =====
public class DiscountStrategyFactory {
    private static final Map<String, DiscountStrategy> strategies = new HashMap<>();
    
    static {
        // 注册策略（可以从配置文件/数据库加载）
        strategies.put("none",       new NoDiscountStrategy());
        strategies.put("90percent",  new NinetyPercentStrategy());
        strategies.put("full100-20", new FullReductionStrategy(
            new BigDecimal("100"), new BigDecimal("20")));
        strategies.put("buy3get1",   new BuyNGetOneFreeStrategy(3));
        strategies.put("vip1",       new VIPDiscountStrategy(1));
        strategies.put("vip5",       new VIPDiscountStrategy(5));
    }
    
    public static DiscountStrategy getStrategy(String type) {
        DiscountStrategy strategy = strategies.get(type);
        if (strategy == null) {
            // 默认无折扣，而不是抛异常
            return strategies.get("none");
        }
        return strategy;
    }
    
    // 支持动态注册新策略（扩展开放，修改关闭）
    public static void register(String type, DiscountStrategy strategy) {
        strategies.put(type, strategy);
    }
}

// 客户端：无任何if-else
public class OrderProcessor {
    public void processOrder(String discountType, List<CartItem> items) {
        // 通过工厂获取策略（无if-else！）
        DiscountStrategy strategy = DiscountStrategyFactory.getStrategy(discountType);
        ShoppingCart cart = new ShoppingCart(strategy);
        items.forEach(item -> cart.addItem(item.getName(), item.getPrice(), item.getQuantity()));
        cart.checkout();
    }
}

class CartItem {
    private String name;
    private BigDecimal price;
    private int quantity;
    public CartItem(String name, BigDecimal price, int qty) {
        this.name = name; this.price = price; this.quantity = qty;
    }
    public String getName() { return name; }
    public BigDecimal getPrice() { return price; }
    public int getQuantity() { return quantity; }
}
```

## 4.2 模板方法模式 Template Method

### 定义与意图

> **定义一个操作中的算法的骨架，而将一些步骤延迟到子类中**。模板方法使得子类可以不改变一个算法的结构即可重定义该算法的某些特定步骤。

**核心**：父类定义流程（骨架），子类实现具体步骤（钩子）。

### 完整代码示例（数据采集处理框架）

```java
// 模板方法抽象类
public abstract class DataMiningTemplate {
    
    // 模板方法：定义数据挖掘的算法骨架（final防止子类覆盖流程）
    public final void mine(String dataSource) {
        System.out.println("=== 开始数据挖掘: " + dataSource + " ===");
        
        // 1. 连接数据源（抽象，子类实现）
        connect(dataSource);
        
        // 2. 提取数据（抽象，子类实现）
        List<String> rawData = extractData();
        
        // 3. 解析数据（抽象，子类实现）
        List<Object> parsedData = parseData(rawData);
        
        // 4. 分析数据（有默认实现，子类可选覆盖 — Hook方法）
        if (needsAnalysis()) { // 钩子：控制是否执行分析
            analyzeData(parsedData);
        }
        
        // 5. 生成报告（具体方法，所有子类共用）
        generateReport(parsedData);
        
        // 6. 关闭连接（具体方法）
        disconnect();
        
        System.out.println("=== 数据挖掘完成 ===");
    }
    
    // 抽象步骤：子类必须实现
    protected abstract void connect(String dataSource);
    protected abstract List<String> extractData();
    protected abstract List<Object> parseData(List<String> rawData);
    
    // 钩子方法（Hook）：子类可选覆盖
    protected boolean needsAnalysis() { return true; } // 默认需要分析
    
    protected void analyzeData(List<Object> data) {
        System.out.println("  [默认分析] 数据量: " + data.size());
    }
    
    // 具体方法：所有子类共用（不需要覆盖）
    protected void generateReport(List<Object> data) {
        System.out.println("  [报告] 处理完成，共 " + data.size() + " 条记录");
    }
    
    protected void disconnect() {
        System.out.println("  [连接] 已关闭");
    }
}

// 具体子类A：MySQL数据挖掘
public class MySQLDataMining extends DataMiningTemplate {
    
    @Override
    protected void connect(String dataSource) {
        System.out.println("  [MySQL] 连接到数据库: " + dataSource);
    }
    
    @Override
    protected List<String> extractData() {
        System.out.println("  [MySQL] 执行SQL查询...");
        return Arrays.asList("row1", "row2", "row3", "row4");
    }
    
    @Override
    protected List<Object> parseData(List<String> rawData) {
        System.out.println("  [MySQL] 解析ResultSet...");
        return rawData.stream()
            .map(s -> (Object)("MySQL_" + s.toUpperCase()))
            .collect(Collectors.toList());
    }
    
    @Override
    protected void analyzeData(List<Object> data) {
        System.out.println("  [MySQL分析] 执行数据库聚合计算: " + data.size() + " 条");
    }
}

// 具体子类B：CSV文件数据挖掘
public class CSVDataMining extends DataMiningTemplate {
    
    @Override
    protected void connect(String dataSource) {
        System.out.println("  [CSV] 打开文件: " + dataSource);
    }
    
    @Override
    protected List<String> extractData() {
        System.out.println("  [CSV] 逐行读取...");
        return Arrays.asList("Alice,30,Engineer", "Bob,25,Designer", "Carol,35,Manager");
    }
    
    @Override
    protected List<Object> parseData(List<String> rawData) {
        System.out.println("  [CSV] 解析CSV行...");
        return rawData.stream()
            .map(line -> (Object)(line.split(",")[0]))  // 只取名字
            .collect(Collectors.toList());
    }
    
    // 覆盖钩子：CSV文件不需要分析步骤
    @Override
    protected boolean needsAnalysis() { return false; }
}

// 使用
public class DataMiningDemo {
    public static void main(String[] args) {
        DataMiningTemplate mySQLMiner = new MySQLDataMining();
        mySQLMiner.mine("jdbc:mysql://localhost:3306/sales");
        
        System.out.println();
        
        DataMiningTemplate csvMiner = new CSVDataMining();
        csvMiner.mine("data/users.csv");
    }
}
```

## 4.3 观察者模式 Observer

### 定义与意图

> **定义对象间的一种一对多的依赖关系**，当一个对象的状态发生改变时，所有依赖于它的对象都得到通知并被自动更新。

**别名**：发布-订阅模式（Publish-Subscribe，但两者有细微区别）。

### 完整代码示例（实时股票行情系统）

```java
import java.util.List;
import java.util.ArrayList;
import java.util.Map;
import java.util.HashMap;

// 观察者接口
public interface StockObserver {
    void update(String stockCode, double price, double changeRate);
}

// 被观察者接口
public interface StockSubject {
    void addObserver(StockObserver observer);
    void removeObserver(StockObserver observer);
    void notifyObservers();
}

// 具体被观察者：股票
public class Stock implements StockSubject {
    private final String code;
    private double price;
    private double prevPrice;
    private final List<StockObserver> observers = new ArrayList<>();
    
    public Stock(String code, double initPrice) {
        this.code = code;
        this.price = initPrice;
        this.prevPrice = initPrice;
    }
    
    @Override
    public void addObserver(StockObserver observer) {
        observers.add(observer);
    }
    
    @Override
    public void removeObserver(StockObserver observer) {
        observers.remove(observer);
    }
    
    @Override
    public void notifyObservers() {
        double changeRate = (price - prevPrice) / prevPrice * 100;
        for (StockObserver observer : observers) {
            observer.update(code, price, changeRate);
        }
    }
    
    // 价格变动 → 自动通知所有观察者
    public void setPrice(double newPrice) {
        this.prevPrice = this.price;
        this.price = newPrice;
        notifyObservers(); // 状态变更后自动推送
    }
    
    public String getCode() { return code; }
    public double getPrice() { return price; }
}

// 具体观察者A：移动端推送
public class MobilePushObserver implements StockObserver {
    private final String userId;
    
    public MobilePushObserver(String userId) {
        this.userId = userId;
    }
    
    @Override
    public void update(String stockCode, double price, double changeRate) {
        String direction = changeRate >= 0 ? "↑" : "↓";
        System.out.printf("[手机推送→%s] %s: %.2f %s%.2f%%%n",
            userId, stockCode, price, direction, Math.abs(changeRate));
    }
}

// 具体观察者B：邮件告警（只在涨跌幅超过5%时告警）
public class EmailAlertObserver implements StockObserver {
    private final String email;
    private final double threshold;
    
    public EmailAlertObserver(String email, double threshold) {
        this.email = email;
        this.threshold = threshold;
    }
    
    @Override
    public void update(String stockCode, double price, double changeRate) {
        if (Math.abs(changeRate) >= threshold) {
            System.out.printf("[邮件告警→%s] 警告！%s 涨跌幅 %.2f%% 超过阈值 %.1f%%%n",
                email, stockCode, changeRate, threshold);
        }
    }
}

// 具体观察者C：自动交易系统
public class AutoTradingObserver implements StockObserver {
    private final Map<String, Double> holdingCosts = new HashMap<>(); // 持仓成本
    
    public void addHolding(String stockCode, double cost) {
        holdingCosts.put(stockCode, cost);
    }
    
    @Override
    public void update(String stockCode, double price, double changeRate) {
        Double cost = holdingCosts.get(stockCode);
        if (cost == null) return;
        
        double profit = (price - cost) / cost * 100;
        if (profit >= 20) {
            System.out.printf("[自动交易] %s 盈利 %.1f%%，触发止盈，自动卖出！%n",
                stockCode, profit);
        } else if (profit <= -10) {
            System.out.printf("[自动交易] %s 亏损 %.1f%%，触发止损，自动卖出！%n",
                stockCode, Math.abs(profit));
        }
    }
}

// 使用
public class StockMarketDemo {
    public static void main(String[] args) {
        Stock appleStock = new Stock("AAPL", 150.0);
        
        // 订阅
        appleStock.addObserver(new MobilePushObserver("user_001"));
        appleStock.addObserver(new MobilePushObserver("user_002"));
        appleStock.addObserver(new EmailAlertObserver("admin@example.com", 5.0));
        
        AutoTradingObserver autoTrader = new AutoTradingObserver();
        autoTrader.addHolding("AAPL", 120.0); // 120元买入
        appleStock.addObserver(autoTrader);
        
        // 价格变动，自动通知所有观察者
        System.out.println("--- 价格变动 ---");
        appleStock.setPrice(153.0); // +2%
        System.out.println();
        appleStack.setPrice(141.0); // -6%（超过告警阈值）
        System.out.println();
        appleStock.setPrice(145.0);
    }
```

## 4.4 责任链模式 Chain of Responsibility

### 定义与意图

> **使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合关系**。将这些对象连成一条链，并沿着这条链传递该请求，直到有一个对象处理它为止。

### 完整代码示例（审批流程）

```java
// 请假申请
public class LeaveRequest {
    private final String employeeName;
    private final int days;
    private final String reason;
    
    public LeaveRequest(String name, int days, String reason) {
        this.employeeName = name;
        this.days = days;
        this.reason = reason;
    }
    
    public String getEmployeeName() { return employeeName; }
    public int getDays() { return days; }
    public String getReason() { return reason; }
    
    @Override
    public String toString() {
        return employeeName + " 申请请假 " + days + " 天，原因: " + reason;
    }
}

// 抽象处理者
public abstract class ApprovalHandler {
    protected ApprovalHandler nextHandler; // 下一个处理者
    protected String handlerName;
    
    public ApprovalHandler(String handlerName) {
        this.handlerName = handlerName;
    }
    
    // 设置下一个处理者（链式方法）
    public ApprovalHandler setNext(ApprovalHandler handler) {
        this.nextHandler = handler;
        return handler; // 返回next，支持链式设置
    }
    
    // 处理请求（模板方法）
    public final void handle(LeaveRequest request) {
        if (canHandle(request)) {
            approve(request);
        } else if (nextHandler != null) {
            System.out.println("[" + handlerName + "] 超出权限，转交上级");
            nextHandler.handle(request);
        } else {
            System.out.println("[" + handlerName + "] 无法处理，请假申请被拒绝");
        }
    }
    
    protected abstract boolean canHandle(LeaveRequest request);
    protected abstract void approve(LeaveRequest request);
}

// 具体处理者A：组长（可审批1天以内）
public class TeamLeaderHandler extends ApprovalHandler {
    public TeamLeaderHandler() { super("组长"); }
    
    @Override
    protected boolean canHandle(LeaveRequest request) {
        return request.getDays() <= 1;
    }
    
    @Override
    protected void approve(LeaveRequest request) {
        System.out.println("[组长] 批准: " + request);
    }
}

// 具体处理者B：部门经理（可审批3天以内）
public class ManagerHandler extends ApprovalHandler {
    public ManagerHandler() { super("部门经理"); }
    
    @Override
    protected boolean canHandle(LeaveRequest request) {
        return request.getDays() <= 3;
    }
    
    @Override
    protected void approve(LeaveRequest request) {
        System.out.println("[部门经理] 批准: " + request);
    }
}

// 具体处理者C：HR总监（可审批7天以内）
public class HRDirectorHandler extends ApprovalHandler {

// 具体处理者D：CEO（可审批任意天数）
public class CEOHandler extends ApprovalHandler {
    public CEOHandler() { super("CEO"); }
    
    @Override
    protected boolean canHandle(LeaveRequest request) {
        return true; // CEO可以审批任意天数
    }

    protected void approve(LeaveRequest request) {
        System.out.println("[CEO] 批准: " + request);

// 组建责任链并使用
public class ApprovalChainDemo {
    public static void main(String[] args) {
        // 组建审批链：组长 → 经理 → HR总监 → CEO
        ApprovalHandler leader = new TeamLeaderHandler();
        ApprovalHandler manager = new ManagerHandler();
        ApprovalHandler hrDirector = new HRDirectorHandler();
        ApprovalHandler ceo = new CEOHandler();
        
        leader.setNext(manager).setNext(hrDirector).setNext(ceo); // 链式组建
        
        // 测试不同天数的请假申请
        leader.handle(new LeaveRequest("张三", 1, "身体不适"));   // 组长处理
        System.out.println();
```

## 4.5 命令模式 Command

### 定义与意图

> **将一个请求封装成一个对象**，从而使你可用不同的请求对客户进行参数化；对请求排队或记录请求日志，以及支持可撤销的操作。

### 完整代码示例（文本编辑器撤销/重做）

```java
import java.util.Deque;
import java.util.ArrayDeque;

// 命令接口
public interface TextCommand {
    void execute();
    void undo();
    String getDescription();
}

// 接收者：文本编辑器
public class TextEditor {
    private StringBuilder content = new StringBuilder();
    private int cursorPosition = 0;
    
    public void insertText(int position, String text) {
        content.insert(position, text);
        cursorPosition = position + text.length();
    }
    
    public void deleteText(int start, int length) {
        content.delete(start, start + length);
        cursorPosition = start;
    }
    
    public String getContent() { return content.toString(); }
    public int getCursorPosition() { return cursorPosition; }
    
    @Override
    public String toString() {
        return "TextEditor{content='" + content + "', cursor=" + cursorPosition + "}";
    }
}

// 具体命令：插入文本
public class InsertTextCommand implements TextCommand {
    private final TextEditor editor;
    private final int position;
    private final String text;
    
    public InsertTextCommand(TextEditor editor, int position, String text) {
        this.editor = editor;
        this.position = position;
        this.text = text;
    }
    
    @Override
    public void execute() {
        editor.insertText(position, text);
    }
    
    @Override
    public void undo() {
        editor.deleteText(position, text.length()); // 撤销：删除刚才插入的文本
    }
    
    @Override
    public String getDescription() {
        return "在位置" + position + "插入: \"" + text + "\"";
    }
}

// 具体命令：删除文本
public class DeleteTextCommand implements TextCommand {
    private final TextEditor editor;
    private final int start;
    private final int length;
    private String deletedText; // 保存被删除的文本（用于撤销）
    
    public DeleteTextCommand(TextEditor editor, int start, int length) {
        this.editor = editor;
        this.start = start;
        this.length = length;
    }
    
    @Override
    public void execute() {
        // 执行前保存被删除的内容
        deletedText = editor.getContent().substring(start, start + length);
        editor.deleteText(start, length);
    }
    
    @Override
    public void undo() {
        editor.insertText(start, deletedText); // 撤销：重新插入被删除的文本
    }
    
    @Override
    public String getDescription() {
        return "删除位置" + start + "起的" + length + "个字符";
    }
}

// 命令调用者：支持撤销/重做
public class CommandHistory {
    private final Deque<TextCommand> undoStack = new ArrayDeque<>();
    private final Deque<TextCommand> redoStack = new ArrayDeque<>();
    
    public void execute(TextCommand command) {
        command.execute();
        undoStack.push(command);
        redoStack.clear(); // 执行新命令后清空重做栈
        System.out.println("[执行] " + command.getDescription());
    }
    
    public void undo() {
        if (undoStack.isEmpty()) {
            System.out.println("[撤销] 没有可撤销的操作");
            return;
        }
        TextCommand command = undoStack.pop();
        command.undo();
        redoStack.push(command);
        System.out.println("[撤销] " + command.getDescription());
    }
    
    public void redo() {
        if (redoStack.isEmpty()) {
            System.out.println("[重做] 没有可重做的操作");
            return;
        }
        TextCommand command = redoStack.pop();
        command.execute();
        undoStack.push(command);
        System.out.println("[重做] " + command.getDescription());
    }
}

// 使用
public class TextEditorDemo {
    public static void main(String[] args) {
        TextEditor editor = new TextEditor();
        CommandHistory history = new CommandHistory();
        
        history.execute(new InsertTextCommand(editor, 0, "Hello"));
        System.out.println("内容: " + editor.getContent());
        
        history.execute(new InsertTextCommand(editor, 5, " World"));
        System.out.println("内容: " + editor.getContent());
        
        history.execute(new DeleteTextCommand(editor, 5, 6));
        System.out.println("内容: " + editor.getContent());
        
        history.undo(); // 撤销删除
        System.out.println("撤销后: " + editor.getContent());
        
        history.undo(); // 撤销第二次插入
        System.out.println("撤销后: " + editor.getContent());
        
        history.redo(); // 重做
        System.out.println("重做后: " + editor.getContent());
    }
}
```

## 4.6 迭代器模式 Iterator

### 定义与意图

> **提供一种方法顺序访问一个聚合对象中的各个元素**，而又不需要暴露该对象的内部表示。

### 完整代码示例

```java
// 自定义迭代器接口
public interface Iterator<E> {
    boolean hasNext();
    E next();
    default void remove() { throw new UnsupportedOperationException(); }
}

// 聚合接口
public interface Aggregate<E> {
    Iterator<E> createIterator();
    int size();
}

// 具体聚合：图书馆书架（内部用数组存储）
public class BookShelf implements Aggregate<Book> {
    private Book[] books;
    private int count = 0;
    
    public BookShelf(int capacity) {
        this.books = new Book[capacity];
    }
    
    public void addBook(Book book) {
        if (count < books.length) {
            books[count++] = book;
        }
    }
    
    public Book getBookAt(int index) {
        return books[index];
    }
    
    @Override
    public int size() { return count; }
    
    @Override
    public Iterator<Book> createIterator() {
        return new BookShelfIterator(this);
    }
    
    // 也可以实现Java标准 Iterable 接口，支持增强for循环
    public java.util.Iterator<Book> iterator() {
        return new java.util.Iterator<Book>() {
            private int cursor = 0;
            
            @Override
            public boolean hasNext() { return cursor < count; }
            
            @Override
            public Book next() {
                if (!hasNext()) throw new java.util.NoSuchElementException();
                return books[cursor++];
            }
        };
    }
}

// 具体迭代器
public class BookShelfIterator implements Iterator<Book> {
    private final BookShelf bookShelf;
    private int cursor = 0;
    
    public BookShelfIterator(BookShelf shelf) {
        this.bookShelf = shelf;
    }
    
    @Override
    public boolean hasNext() {
        return cursor < bookShelf.size();
    }
    
    @Override
    public Book next() {
        if (!hasNext()) throw new RuntimeException("没有更多书了");
        return bookShelf.getBookAt(cursor++);
    }
}

public class Book {
    private String title;
    private String author;
    
    public Book(String title, String author) {
        this.title = title;
        this.author = author;
    }
    
    @Override
    public String toString() { return "《" + title + "》 by " + author; }
}

// 客户端：通过迭代器访问，不需要知道内部是数组还是链表
public class LibraryDemo {
    public static void main(String[] args) {
        BookShelf shelf = new BookShelf(10);
        shelf.addBook(new Book("设计模式", "GoF"));
        shelf.addBook(new Book("重构", "Martin Fowler"));
        shelf.addBook(new Book("代码整洁之道", "Robert Martin"));
        shelf.addBook(new Book("Effective Java", "Joshua Bloch"));
        
        // 使用自定义迭代器
        Iterator<Book> it = shelf.createIterator();
        System.out.println("=== 遍历书架 ===");
        while (it.hasNext()) {
            System.out.println(it.next());
        }
    }
}
```

## 4.7 中介者模式 Mediator

### 定义与意图

> **用一个中介对象来封装一系列的对象交互**，中介者使各对象不需要显式地相互引用，从而使其耦合松散。

**解决问题**：当N个对象互相引用时，复杂度是O(N²)。中介者将其降为O(N)。

### 完整代码示例（聊天室）

```java
// 中介者接口
public interface ChatMediator {
    void sendMessage(String message, ChatUser sender);
    void addUser(ChatUser user);
    void removeUser(ChatUser user);
}

// 聊天用户（同事类）
public abstract class ChatUser {
    protected final ChatMediator mediator;
    protected final String name;
    
    public ChatUser(ChatMediator mediator, String name) {
        this.mediator = mediator;
        this.name = name;
    }
    
    public abstract void send(String message);
    public abstract void receive(String message, String from);
    
    public String getName() { return name; }
}

// 具体中介者：聊天室
public class ChatRoom implements ChatMediator {
    private final String roomName;
    private final List<ChatUser> users = new ArrayList<>();
    private final List<String> messageHistory = new ArrayList<>();
    
    public ChatRoom(String roomName) {
        this.roomName = roomName;
    }
    
    @Override
    public void addUser(ChatUser user) {
        users.add(user);
        System.out.println("[" + roomName + "] " + user.getName() + " 加入聊天室");
        // 通知其他用户
        sendSystemMessage(user.getName() + " 加入了聊天室");
    }
    
    @Override
    public void removeUser(ChatUser user) {
        users.remove(user);
        sendSystemMessage(user.getName() + " 离开了聊天室");
    }
    
    @Override
    public void sendMessage(String message, ChatUser sender) {
        String formattedMsg = "[" + sender.getName() + "]: " + message;
        messageHistory.add(formattedMsg);
        
        // 转发给除发送者外的所有用户（中介者协调！）
        for (ChatUser user : users) {
            if (user != sender) {
                user.receive(message, sender.getName());
            }
        }
    }
    
    private void sendSystemMessage(String message) {
        for (ChatUser user : users) {
            user.receive(message, "系统");
        }
    }
    
    public List<String> getHistory() {
        return Collections.unmodifiableList(messageHistory);
    }
}

// 具体用户
public class ConcreteUser extends ChatUser {
    
    public ConcreteUser(ChatMediator mediator, String name) {
        super(mediator, name);
    }
    
    @Override
    public void send(String message) {
        System.out.println("[" + name + " 发送]: " + message);
        mediator.sendMessage(message, this); // 通过中介者发送
    }
    
    @Override
    public void receive(String message, String from) {
        System.out.println("  [" + name + " 收到 from " + from + "]: " + message);
    }
}

// 演示
public class ChatRoomDemo {
    public static void main(String[] args) {
        ChatRoom room = new ChatRoom("Java技术交流群");
        
        ChatUser alice = new ConcreteUser(room, "Alice");
        ChatUser bob = new ConcreteUser(room, "Bob");
        ChatUser carol = new ConcreteUser(room, "Carol");
        
        room.addUser(alice);
        room.addUser(bob);
        room.addUser(carol);
        
        System.out.println();
        alice.send("大家好！有人了解设计模式吗？");
        System.out.println();
        bob.send("我在学习，有什么问题？");
        System.out.println();
        carol.send("我推荐《Head First 设计模式》这本书");
    }
}
```

## 4.8 备忘录模式 Memento

### 定义与意图

> **在不破坏封装性的前提下，捕获一个对象的内部状态，并在该对象之外保存这个状态**，这样以后就可以将该对象恢复到原先保存的状态。

### 完整代码示例（游戏存档）

```java
// 备忘录：保存游戏状态（不对外暴露内部结构）
public class GameMemento {
    private final int level;
    private final int hp;
    private final int exp;
    private final String location;
    private final long timestamp;
    
    // 包级私有构造器，只有同包的 GameCharacter 可以创建
    GameMemento(int level, int hp, int exp, String location) {
        this.level = level;
        this.hp = hp;
        this.exp = exp;
        this.location = location;
        this.timestamp = System.currentTimeMillis();
    }
    
    // 只提供读取方法，防止外部修改
    int getLevel() { return level; }
    int getHp() { return hp; }
    int getExp() { return exp; }
    String getLocation() { return location; }
    
    @Override
    public String toString() {
        return String.format("存档[Lv%d, HP:%d, EXP:%d, 位置:%s]", level, hp, exp, location);
    }
}

// 原发器：游戏角色
public class GameCharacter {
    private int level;
    private int hp;
    private int exp;
    private String location;
    
    public GameCharacter(String startLocation) {
        this.level = 1;
        this.hp = 100;
        this.exp = 0;
        this.location = startLocation;
    }
    
    // 创建备忘录（保存状态）
    public GameMemento save() {
        return new GameMemento(level, hp, exp, location);
    }
    
    // 从备忘录恢复状态
    public void restore(GameMemento memento) {
        this.level = memento.getLevel();
        this.hp = memento.getHp();
        this.exp = memento.getExp();
        this.location = memento.getLocation();
    }
    
    public void gainExp(int amount) {
        this.exp += amount;
        if (this.exp >= level * 100) {
            this.level++;
            this.hp = 100 + level * 20;
            System.out.println("升级！当前等级: " + level);
        }
    }
    
    public void takeDamage(int damage) {
        this.hp = Math.max(0, this.hp - damage);
    }
    
    public void moveTo(String location) {
        this.location = location;
    }
    
    @Override
    public String toString() {
        return String.format("角色[Lv%d, HP:%d/%d, EXP:%d, 位置:%s]",
            level, hp, 100 + level * 20, exp, location);
    }
}

// 管理者：存档管理器
public class GameSaveManager {
    private final Deque<GameMemento> saves = new ArrayDeque<>();
    private static final int MAX_SAVES = 5; // 最多保留5个存档
    
    public void save(GameCharacter character) {
        GameMemento memento = character.save();
        saves.push(memento);
        System.out.println("[存档] " + memento);
        
        // 限制存档数量
        while (saves.size() > MAX_SAVES) {
            saves.pollLast();
        }
    }
    
    public GameMemento getLatestSave() {
        return saves.isEmpty() ? null : saves.peek();
    }
    
    public void restore(GameCharacter character) {
        if (saves.isEmpty()) {
            System.out.println("[读档] 没有存档可以恢复");
            return;
        }
        GameMemento memento = saves.pop();
        character.restore(memento);
        System.out.println("[读档] 恢复到: " + memento);
    }
    
    public void listSaves() {
        System.out.println("[存档列表] 共 " + saves.size() + " 个存档:");
        saves.forEach(m -> System.out.println("  " + m));
    }
}

// 使用
public class GameDemo {
    public static void main(String[] args) {
        GameCharacter hero = new GameCharacter("新手村");
        GameSaveManager saveManager = new GameSaveManager();
        
        System.out.println("初始状态: " + hero);
        saveManager.save(hero); // 存档1
        
        hero.moveTo("黑暗森林");
        hero.gainExp(80);
        System.out.println("打怪后: " + hero);
        saveManager.save(hero); // 存档2
        
        hero.moveTo("Boss房间");
        hero.takeDamage(90);
        System.out.println("被Boss打了: " + hero);
        
        // 读档回到上一个存档
        saveManager.restore(hero);
        System.out.println("读档后: " + hero);
    }
}
```

## 4.9 状态模式 State

### 定义与意图

> **允许对象在内部状态改变时改变它的行为**，对象看起来好像修改了它的类。

**解决问题**：消除大量的状态判断 if-else，将每个状态封装为独立类。

### 完整代码示例（订单状态机）

```java
// 状态接口
public interface OrderState {
    void pay(OrderContext order);
    void ship(OrderContext order);
    void receive(OrderContext order);
    void cancel(OrderContext order);
    String getStateName();
}

// Context：订单（维护当前状态引用）
public class OrderContext {
    private OrderState currentState;
    private final Long orderId;
    private String history = "";
    
    // 初始状态：待支付
    public OrderContext(Long orderId) {
        this.orderId = orderId;
        this.currentState = new PendingPaymentState();
        log("订单创建");
    }
    
    // 委托给当前状态处理
    public void pay() { currentState.pay(this); }
    public void ship() { currentState.ship(this); }
    public void receive() { currentState.receive(this); }
    public void cancel() { currentState.cancel(this); }
    
    // 状态转换（由State类调用）
    public void setState(OrderState state) {
        log("状态变更: " + currentState.getStateName() + " → " + state.getStateName());
        this.currentState = state;
    }
    
    public OrderState getCurrentState() { return currentState; }
    public Long getOrderId() { return orderId; }
    
    private void log(String event) {
        history += "\n  [" + event + "]";
    }
    
    public void printHistory() {
        System.out.println("订单 #" + orderId + " 操作历史:" + history);
    }
}

// 具体状态：待支付
public class PendingPaymentState implements OrderState {

}

}

// 具体状态：已发货
public class ShippedState implements OrderState {
```

## 4.11 解释器模式 Interpreter

### 定义与意图

> **给定一个语言，定义它的文法的一种表示，并定义一个解释器**，这个解释器使用该表示来解释语言中的句子。

**适用场景**：SQL解析、正则表达式、脚本引擎、规则引擎。

### 完整代码示例（简单规则引擎）

```java
// 抽象表达式
public interface Expression {
    boolean interpret(Map<String, Object> context);
    String describe();
}

// 终结符：变量比较
public class VariableComparatorExpression implements Expression {
    private final String variable;
    private final String operator;
    private final Object value;
    
    public VariableComparatorExpression(String variable, String operator, Object value) {
        this.variable = variable;
        this.operator = operator;
        this.value = value;
    }
    
    @Override
    public boolean interpret(Map<String, Object> context) {
        Object actualValue = context.get(variable);
        if (actualValue == null) return false;
        
        switch (operator) {
            case "==": return actualValue.equals(value);
            case "!=": return !actualValue.equals(value);
            case ">": return toDouble(actualValue) > toDouble(value);
            case "<": return toDouble(actualValue) < toDouble(value);
            case ">=": return toDouble(actualValue) >= toDouble(value);
            case "<=": return toDouble(actualValue) <= toDouble(value);
            default: throw new IllegalArgumentException("未知操作符: " + operator);
        }
    }
    
    private double toDouble(Object obj) {
        return obj instanceof Number ? ((Number)obj).doubleValue() : 0.0;
    }
    
    @Override
    public String describe() {
        return variable + " " + operator + " " + value;
    }
}

// 非终结符：AND表达式
public class AndExpression implements Expression {
    private final Expression left;
    private final Expression right;
    
    public AndExpression(Expression left, Expression right) {
        this.left = left;
        this.right = right;
    }
    
    @Override
    public boolean interpret(Map<String, Object> context) {
        return left.interpret(context) && right.interpret(context);
    }
    
    @Override
    public String describe() {
        return "(" + left.describe() + " AND " + right.describe() + ")";
    }
}

// 非终结符：OR表达式
public class OrExpression implements Expression {
    private final Expression left;
    private final Expression right;
    
    public OrExpression(Expression left, Expression right) {
        this.left = left;
        this.right = right;
    }
    
    @Override
    public boolean interpret(Map<String, Object> context) {
        return left.interpret(context) || right.interpret(context);
    }
    
    @Override
    public String describe() {
        return "(" + left.describe() + " OR " + right.describe() + ")";
    }
}

// 非终结符：NOT表达式
public class NotExpression implements Expression {
    private final Expression expression;
    
    public NotExpression(Expression expression) {
        this.expression = expression;
    }
    
    @Override
    public boolean interpret(Map<String, Object> context) {
        return !expression.interpret(context);
    }
    
    @Override
    public String describe() {
        return "NOT(" + expression.describe() + ")";
    }
}

// 规则引擎（使用解释器）
public class RuleEngine {
    public static void main(String[] args) {
        // 构建规则：年龄>=18 AND (会员等级=="VIP" OR 余额>=1000)
        Expression rule = new AndExpression(
            new VariableComparatorExpression("age", ">=", 18),
            new OrExpression(
                new VariableComparatorExpression("memberLevel", "==", "VIP"),
                new VariableComparatorExpression("balance", ">=", 1000.0)
            )
        );
        
        System.out.println("规则: " + rule.describe());
        
        // 测试用户1：18岁VIP会员
        Map<String, Object> user1 = new HashMap<>();
        user1.put("age", 20);
        user1.put("memberLevel", "VIP");
        user1.put("balance", 500.0);
        System.out.println("用户1符合规则: " + rule.interpret(user1)); // true
        
        // 测试用户2：16岁普通用户，余额2000
        Map<String, Object> user2 = new HashMap<>();
        user2.put("age", 16);
        user2.put("memberLevel", "Normal");
        user2.put("balance", 2000.0);
        System.out.println("用户2符合规则: " + rule.interpret(user2)); // false（未成年）
        
        // 测试用户3：25岁普通用户，余额1500
        Map<String, Object> user3 = new HashMap<>();
        user3.put("age", 25);
        user3.put("memberLevel", "Normal");
        user3.put("balance", 1500.0);
        System.out.println("用户3符合规则: " + rule.interpret(user3)); // true
    }
```

```java
/**

 * - 策略模式：不同促销算法（满减/折扣/买赠）

// 促销策略接口（Strategy）
public interface PromotionStrategy {

    String name();

// 满减策略
@Component("fullReduction")
public class FullReductionPromotion implements PromotionStrategy {
    @Override
    public BigDecimal apply(BigDecimal price, Map<String, Object> params) {
        BigDecimal threshold = (BigDecimal) params.getOrDefault("threshold", BigDecimal.ZERO);
        BigDecimal reduction = (BigDecimal) params.getOrDefault("reduction", BigDecimal.ZERO);
        if (price.compareTo(threshold) >= 0) {
            return price.subtract(reduction);
        }
        return price;
    }
    
    @Override
    public String name() { return "满减活动"; }
}

// 折扣策略
@Component("discount")
public class DiscountPromotion implements PromotionStrategy {
    @Override
    public BigDecimal apply(BigDecimal price, Map<String, Object> params) {
        BigDecimal rate = (BigDecimal) params.getOrDefault("rate", BigDecimal.ONE);
        return price.multiply(rate).setScale(2, RoundingMode.HALF_UP);
    }
    
    @Override
    public String name() { return "折扣活动"; }
}

// 策略工厂（Factory + Strategy）
@Component
public class PromotionStrategyFactory {
    // Spring自动注入所有策略（Map的key是Bean名称）
    @Autowired
    private Map<String, PromotionStrategy> strategies;
    
    public PromotionStrategy getStrategy(String type) {
        PromotionStrategy strategy = strategies.get(type);
        if (strategy == null) {
            throw new IllegalArgumentException("未知促销类型: " + type);
        }
        return strategy;
    }
}

// 促销处理模板（Template Method）
public abstract class PromotionProcessor {
    
    // 模板方法：促销处理骨架
    public final PromotionResult process(Order order, PromotionActivity activity) {
        // 1. 校验
        if (!validate(order, activity)) {
            return PromotionResult.invalid("促销活动不适用");
        }
        // 2. 计算优惠
        BigDecimal discount = calculate(order, activity);
        // 3. 记录日志
        record(order, activity, discount);
        // 4. 返回结果
        return PromotionResult.success(discount, activity.getName());
    }
    
    protected abstract boolean validate(Order order, PromotionActivity activity);
    protected abstract BigDecimal calculate(Order order, PromotionActivity activity);
    
    protected void record(Order order, PromotionActivity activity, BigDecimal discount) {
        System.out.printf("[促销] 订单%s 使用[%s] 优惠¥%.2f%n",
            order.getId(), activity.getName(), discount);
    }
}

// 责任链：优惠叠加处理
public abstract class CouponHandler {
    protected CouponHandler next;
    
    public CouponHandler setNext(CouponHandler next) {
        this.next = next;
        return next;
    }
    
    public abstract BigDecimal apply(BigDecimal price, List<Coupon> coupons);
}

// 平台券处理
public class PlatformCouponHandler extends CouponHandler {
    @Override
    public BigDecimal apply(BigDecimal price, List<Coupon> coupons) {
        Optional<Coupon> coupon = coupons.stream()
            .filter(c -> "platform".equals(c.getType()))
            .findFirst();
        
        BigDecimal result = coupon.map(c -> price.subtract(c.getValue())).orElse(price);
        
        if (next != null) {
            return next.apply(result, coupons);
        }
        return result;
    }
}

// 会员折扣处理
public class MemberDiscountHandler extends CouponHandler {
    @Override
    public BigDecimal apply(BigDecimal price, List<Coupon> coupons) {

        BigDecimal memberPrice = price.multiply(new BigDecimal("0.95"))
            .setScale(2, RoundingMode.HALF_UP);
        System.out.println("[会员折扣] 9.5折: " + price + " → " + memberPrice);
        
        if (next != null) {
            return next.apply(memberPrice, coupons);
        }
        return memberPrice;
    }
}
```

## 5.2 工作流引擎（状态机+命令+观察者）

```java
/**
 * 工作流状态机：
 * - 状态模式：工单各个状态（待处理/处理中/待审核/已完成）
 * - 命令模式：工单操作（分配/处理/提交审核/审批）支持撤销
 * - 观察者：状态变更时自动通知相关人员
 */

public enum WorkflowState {
    PENDING,        // 待处理
    IN_PROGRESS,    // 处理中
    PENDING_REVIEW, // 待审核
    APPROVED,       // 审核通过
    REJECTED,       // 审核拒绝
    CLOSED          // 已关闭
}

@Entity
public class WorkflowTask {
    @Id
    private Long id;
    
    @Enumerated(EnumType.STRING)
    private WorkflowState state = WorkflowState.PENDING;
    
    private String title;
    private String assignee;
    private String content;
    private LocalDateTime updatedAt;
    
    // 状态转换（结合状态模式）
    public void assign(String assignee) {
        if (state != WorkflowState.PENDING) {
            throw new IllegalStateException("只有待处理状态才能分配");
        }
        this.assignee = assignee;
        this.state = WorkflowState.IN_PROGRESS;
        this.updatedAt = LocalDateTime.now();
    }
    
    public void submitForReview() {
        if (state != WorkflowState.IN_PROGRESS) {
            throw new IllegalStateException("只有处理中状态才能提交审核");
        }
        this.state = WorkflowState.PENDING_REVIEW;
        this.updatedAt = LocalDateTime.now();
    }
    
    public void approve() {
        if (state != WorkflowState.PENDING_REVIEW) {
            throw new IllegalStateException("只有待审核状态才能审批");
        }
        this.state = WorkflowState.APPROVED;
        this.updatedAt = LocalDateTime.now();
    }
    
    public void reject(String reason) {
        if (state != WorkflowState.PENDING_REVIEW) {
            throw new IllegalStateException("只有待审核状态才能拒绝");
        }
        this.state = WorkflowState.REJECTED;
        this.content = reason;
        this.updatedAt = LocalDateTime.now();
    }
}

// 工作流服务（结合命令+观察者）
@Service
@Transactional
public class WorkflowService {
    @Autowired
    private WorkflowTaskRepository repository;

        WorkflowState prevState = task.getState();
        task.assign(assignee);
        repository.save(task);
        
        // 发布状态变更事件（观察者模式）
        eventPublisher.publishEvent(new WorkflowStateChangedEvent(
            this, task, prevState, task.getState()));

    // 其他操作类似...
}

                break;
```

## 5.4 数据导出框架（模板方法+策略+装饰器）

```java
/**
 * 通用数据导出框架：
 * - 模板方法：定义导出流程（查询→转换→写入→通知）
 * - 策略：不同导出格式（Excel/CSV/PDF/JSON）
 * - 装饰器：增加加密/压缩/水印功能
 */

// 导出格式策略
public interface ExportStrategy<T> {
    byte[] export(List<T> data, ExportConfig config);
    String getFileExtension();
    String getMimeType();
}

// CSV导出策略
public class CSVExportStrategy<T> implements ExportStrategy<T> {
    @Override
    public byte[] export(List<T> data, ExportConfig config) {
        StringBuilder sb = new StringBuilder();
        // 写表头
        sb.append(String.join(",", config.getHeaders())).append("\n");
        // 写数据
        for (T row : data) {
            sb.append(convertToCSVRow(row, config)).append("\n");
        }
        return sb.toString().getBytes(StandardCharsets.UTF_8);
    }
    
    private String convertToCSVRow(T row, ExportConfig config) {
        // 反射获取字段值，转为CSV行
        return config.getFieldExtractor().apply(row).stream()
            .map(v -> "\"" + (v == null ? "" : v.toString().replace("\"", "\"\"")) + "\"")
            .collect(Collectors.joining(","));
    }
    
    @Override
    public String getFileExtension() { return "csv"; }
    
    @Override
    public String getMimeType() { return "text/csv"; }
}

// 导出模板
public abstract class DataExportTemplate<T> {
    
    private final ExportStrategy<T> strategy;
    
    public DataExportTemplate(ExportStrategy<T> strategy) {
        this.strategy = strategy;
    }
    
    // 模板方法
    public final ExportResult export(ExportRequest request) {
        // 1. 验证权限
        validatePermission(request);
        
        // 2. 查询数据
        List<T> data = fetchData(request);
        
        // 3. 转换/过滤数据
        data = transformData(data, request);
        
        // 4. 执行导出
        ExportConfig config = buildConfig(request);
        byte[] content = strategy.export(data, config);
        
        // 5. 后处理（如加水印、压缩）
        content = postProcess(content, request);
        
        // 6. 保存/上传文件
        String fileUrl = saveFile(content, strategy.getFileExtension(), request);
        
        // 7. 通知（如发送邮件）
        notifyComplete(request, fileUrl);
        
        return new ExportResult(fileUrl, content.length, data.size());
    }
    
    protected abstract void validatePermission(ExportRequest request);
    protected abstract List<T> fetchData(ExportRequest request);
    protected abstract ExportConfig buildConfig(ExportRequest request);
    
    // 可选覆盖的钩子
    protected List<T> transformData(List<T> data, ExportRequest request) { return data; }
    protected byte[] postProcess(byte[] content, ExportRequest request) { return content; }
    protected String saveFile(byte[] content, String ext, ExportRequest request) {
        String filename = "export_" + System.currentTimeMillis() + "." + ext;
        System.out.println("[导出] 文件保存: " + filename + " (" + content.length + " bytes)");
        return "/exports/" + filename;
    }
    protected void notifyComplete(ExportRequest request, String fileUrl) {
        System.out.println("[导出] 完成通知 → " + request.getNotifyEmail() + ": " + fileUrl);
    }
}

// 订单导出实现
public class OrderExportTemplate extends DataExportTemplate<Order> {
    
    @Autowired
    private OrderRepository orderRepository;
    
    public OrderExportTemplate(ExportStrategy<Order> strategy) {
        super(strategy);
    }
    
    @Override
    protected void validatePermission(ExportRequest request) {
        // 检查用户是否有导出权限
        System.out.println("[权限校验] 用户: " + request.getUserId());
    }
    
    @Override
    protected List<Order> fetchData(ExportRequest request) {
        return orderRepository.findByDateRange(request.getStartDate(), request.getEndDate());
    }
    
    @Override
    protected ExportConfig buildConfig(ExportRequest request) {
        return ExportConfig.builder()
            .headers(Arrays.asList("订单号", "用户", "金额", "状态", "创建时间"))
            .fieldExtractor(order -> Arrays.asList(
                order.getId().toString(),
                order.getUserName(),
                order.getAmount().toString(),
                order.getStatus().name(),
                order.getCreatedAt().toString()
            ))
            .build();
    }
}
```

## Q1：单例模式有哪些实现方式？各自的优缺点？DCL为什么需要volatile？

**答**：
单例的8种实现见Part 2.1节，核心考点是DCL（双重检查锁）：

```java
private static volatile Singleton instance; // volatile关键
```

**volatile的必要性**：`new Singleton()` 在JVM中分3步：
1. 分配堆内存
2. 初始化对象（调用构造器）
3. 将引用指向内存地址

JVM和CPU可能对步骤2、3进行**指令重排序**，导致另一个线程在对象初始化完成之前就拿到了非null引用，访问未完全初始化的对象会导致NPE。`volatile` 的**写屏障**（Write Barrier）阻止了2和3的重排序。

**最推荐**：枚举单例（防反射+防反序列化）或静态内部类（延迟加载+线程安全）。

## Q2：工厂方法和抽象工厂有什么区别？

| 比较项 | 工厂方法 | 抽象工厂 |
|--------|---------|---------|
| 创建对象数 | 一种产品 | 一族产品 |
| 扩展方式 | 新增工厂子类 | 新增产品族工厂 |
| 关注点 | 单个产品的创建 | 多个相关产品的协同 |
| 典型场景 | Logger工厂 | 跨平台UI工厂 |

**记忆**：工厂方法是"一个工厂生产一种产品"，抽象工厂是"一个工厂生产一套产品"。

## Q3：代理模式和装饰器模式有什么区别？

```
代理模式：
  - 目的：控制访问（不暴露真实对象/添加访问控制）
  - 代理通常自己创建被代理对象（客户端不关心）
  - 重点：对访问的控制
  - 例子：Spring AOP、权限控制、远程代理

装饰器模式：
  - 目的：动态增强功能
  - 装饰器由客户端传入被装饰对象
  - 重点：功能的扩展和组合
  - 例子：Java IO流、咖啡+配料
```

**本质区别**：代理关心**谁可以访问**，装饰器关心**怎么增强功能**。

## Q4：观察者模式和发布-订阅模式有什么区别？

```
观察者模式（直接通信）：
  Subject ──直接调用──> Observer
  观察者直接注册到Subject，存在耦合

发布-订阅模式（通过消息中间件）：
  Publisher ──发布──> EventBus/MQ ──推送──> Subscriber
  发布者和订阅者完全解耦，通过中间件通信

例子：
  观察者：JDK的 java.util.Observable
  发布-订阅：Spring的 ApplicationEventPublisher、Kafka、RabbitMQ
```

## Q5：策略模式如何消除if-else？

**核心做法**：用Map替代switch/if-else，将类型映射到策略对象。

```java
// 反例（难以扩展）
if (type.equals("alipay")) { /* 支付宝逻辑 */ }
else if (type.equals("wechat")) { /* 微信逻辑 */ }
else if (type.equals("card")) { /* 银行卡逻辑 */ }

// 正例（策略+工厂）
Map<String, PayStrategy> strategies = new HashMap<>();
strategies.put("alipay",  new AlipayStrategy());
strategies.put("wechat",  new WechatStrategy());
strategies.put("card",    new CardStrategy());

// 使用：无if-else
PayStrategy strategy = strategies.get(type);
strategy.pay(amount);
```

**Spring中更优雅的实现**：
```java
// 每个策略Bean用@Component命名
@Component("alipay")
public class AlipayStrategy implements PayStrategy { ... }

// 自动注入所有策略到Map
@Autowired
Map<String, PayStrategy> strategies; // key=Bean名称
```

## Q6：什么是模板方法模式？Spring中哪里使用了？

模板方法在父类中定义算法骨架（final方法），将变化的步骤留给子类实现。

**Spring中的典型应用**：
- `JdbcTemplate`：骨架（获取连接→执行SQL→处理结果→释放连接），变化的部分由回调实现
- `AbstractBeanFactory.getBean()`：Bean获取的骨架
- `HttpServlet.service()`：分发到doGet/doPost
- `AbstractList`：基于get/size实现迭代等基本方法

## Q7：责任链模式和策略模式有什么区别？

```
策略模式：
  - 从一组算法中选一个执行
  - 互斥，只有一个策略被执行
  - 例：支付方式选择

责任链模式：
  - 请求依次经过链上的处理者
  - 可以多个处理者都处理（或找到第一个处理的为止）
  - 例：Filter链、审批流程
```

## Q8：装饰器模式和继承有什么区别？为什么装饰器更好？

**继承的缺点**：
1. 静态，编译期确定，运行时无法改变
2. 类爆炸：N种咖啡 × M种配料 = N×M个类
3. 破坏封装性

**装饰器的优点**：
1. 动态，运行时自由组合
2. 类数量：N+M个类搞定所有组合
3. 遵循开闭原则：添加新配料只需新类

```java
// 装饰器动态组合（运行时决定）
Beverage drink = new Espresso();           // 基础
drink = new Milk(drink);                   // 加牛奶
if (userWantsWhip) drink = new Whip(drink); // 根据条件加奶泡
if (isVIP) drink = new ExtraShot(drink);   // VIP加一份浓缩
```

## Q9：什么是外观模式？和适配器有什么区别？

| 比较项 | 外观模式 | 适配器模式 |

| 目的 | 简化复杂接口 | 转换不兼容接口 |
| 接口数量 | 多变一（合并） | 一变一（转换） |
| 新增接口 | 是 | 是 |
| 修改接口 | 不修改子系统 | 不修改被适配者 |
| 典型场景 | JdbcTemplate封装JDBC | InputStreamReader |

**记忆**：外观是"遥控器"，适配器是"转换插头"。

## Q10：单例模式如何防止反射和反序列化攻击？

```java
// 反射攻击：
Constructor<Singleton> constructor = Singleton.class.getDeclaredConstructor();
constructor.setAccessible(true);
Singleton s2 = constructor.newInstance(); // 创建了第二个实例！

// 防御方法1：构造器中加判断
private Singleton() {
    if (INSTANCE != null) {
        throw new IllegalStateException("禁止通过反射创建实例！");
    }
}

// 防御方法2：使用枚举（枚举天然防反射和反序列化）
public enum Singleton { INSTANCE; }

// 反序列化攻击防御（非枚举实现）：
protected Object readResolve() throws ObjectStreamException {
    return INSTANCE; // 反序列化时返回已有实例
}
```

## Q11：Spring AOP用的是什么代理？如何选择？

Spring AOP的代理选择规则（Spring Boot 2.x后）：
```
1. 目标类实现了接口 → 默认使用 JDK 动态代理
2. 目标类没有接口 → 使用 CGLIB 代理
3. @EnableAspectJAutoProxy(proxyTargetClass=true) → 强制全部使用 CGLIB

JDK代理创建的对象是接口的实现
CGLIB代理创建的对象是目标类的子类
```

**注意**：CGLIB不能代理final类和final方法。

## Q12：建造者模式和构造器模式的区别？什么时候用建造者？

当构造器参数**超过4个**，或有**多个可选参数**时，优先考虑建造者模式：

```java
new Pizza.Builder("large")
```

## Q13：什么是享元模式？JDK中哪些地方用到了？

享元模式通过**对象复用**减少内存消耗，核心是**对象池/缓存**。

**JDK中的应用**：
- `Integer.valueOf(-128~127)`：缓存池
- `String.intern()`：字符串常量池
- `Boolean.TRUE` / `Boolean.FALSE`：两个静态实例
- `Character.valueOf(0~127)`：字符缓存

**典型面试陷阱**：
```java
Integer a = 100, b = 100; System.out.println(a == b); // true（缓存）
Integer c = 200, d = 200; System.out.println(c == d); // false（超出缓存）
```

**适用场景**：

## Q15：什么场景下不应该使用设计模式？

1. **过度设计**：简单的CRUD不需要工厂/策略
2. **强行套用**：问题本身不符合模式解决的场景
3. **团队理解成本高**：团队不熟悉，维护成本反而增加
4. **性能关键路径**：反射（JDK代理/解释器）在高频调用时影响性能

**原则**：设计模式是解决特定问题的工具，不是炫技工具。

## Q16：命令模式的核心价值是什么？

1. **解耦**：调用者和执行者之间无直接依赖
2. **撤销/重做**：保存命令历史，支持Ctrl+Z
3. **日志/审计**：记录所有命令
4. **队列/延迟执行**：命令可以排队
5. **批量操作**：宏命令（多个命令组合）

**实际应用**：数据库事务、消息队列（消息本身就是命令）、IDE的操作历史。

## Q20：如何在实际项目中选择合适的设计模式？

**思路框架**：

```
1. 识别变化点
   - 什么会变化？（算法、对象创建方式、交互方式）
   - 变化的频率？（频繁变化 → 更需要模式）

2. 确定关系
   - 对象创建问题 → 创建型（工厂/建造者/单例）
   - 类结构问题 → 结构型（适配器/装饰器/代理）
   - 行为协作问题 → 行为型（策略/观察者/责任链）

3. 评估成本
   - 引入模式的复杂度是否值得？
   - 团队是否熟悉？
   - 未来扩展需求是否真实存在？

4. 优先原则
   - 遵循七大设计原则（OCP/DIP/SRP...）
   - 优先组合而非继承
   - 面向接口编程
```

## Q22：什么是双重分派？访问者模式如何利用它？

**单一分派**（Java默认）：方法调用由**接收对象**的运行时类型决定。

**双重分派**：方法调用由**两个**对象的运行时类型共同决定。

Java本身不支持双重分派，访问者模式通过**两次方法调用**模拟：

```java
// 第一次分派：由element的类型决定调用哪个accept()
element.accept(visitor);

// 第二次分派：在accept()内部，由visitor的类型决定调用哪个visit()
public void accept(Visitor v) { v.visit(this); } // 第二次分派
```

这就是为什么访问者模式需要每个元素类都实现 `accept()`。

| 谁保存状态 | 独立的Memento对象 | 克隆体本身 |
| 恢复 | 可以恢复到历史状态 | 克隆后各自独立 |
| 关系 | Memento通常用clone实现 | 无需考虑恢复 |
